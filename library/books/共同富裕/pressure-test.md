# 共同富裕 — 压力测试 (阶段 4 产出)

## 测试方法

对每个 skill 设计测试 prompt，包含三类：
- **应调用**: 场景明确匹配 skill 的触发条件
- **不应调用 (诱饵)**: 场景看似相关但实际不匹配
- **边界模糊**: 场景在边界上，需要判断

---

## 1. gyfz-cake-balance

### 应调用
- "我们国家该先发展还是先分配？" → ✓ 应激活
- "企业利润该用于扩张还是分红？" → ✓ 应激活

### 不应调用 (诱饵)
- "今天该加班还是休息？" → ✗ 个人决策，非发展与分配权衡
- "这个技术方案好不好？" → ✗ 无分配维度

### 边界模糊
- "家庭收入该投资还是消费？" → ✓ 如果涉及"发展vs分配"逻辑

---

## 2. gyfz-three-gaps-diagnosis

### 应调用
- "为什么中国区域发展不平衡？" → ✓ 应激活
- "城乡收入差距的根源是什么？" → ✓ 应激活

### 不应调用 (诱饵)
- "为什么我比同事工资低？" → ✗ 个体差异，非结构化差距
- "这个行业的利润率高吗？" → ✗ 单一行业，非差距诊断

### 边界模糊
- "垄断行业和竞争行业收入差距合理吗？" → ✓ 行业差别诊断

---

## 3. gyfz-three-distribution

### 应调用
- "税收、社保、慈善该怎么协调？" → ✓ 应激活
- "企业薪酬体系该怎么设计？" → ✓ 应激活

### 不应调用 (诱饵)
- "该不该对富人多征税？" → ✗ 单一政策，非三层协调
- "慈善该怎么做？" → ✗ 单一层次，非三层设计

### 边界模糊
- "三次分配会不会变成摊派？" → ✓ 涉及三层边界问题

---

## 4. gyfz-gradual-reform

### 应调用
- "这个改革该一步到位还是分步走？" → ✓ 应激活
- "共同富裕的时间表该怎么定？" → ✓ 应激活

### 不应调用 (诱饵)

- "紧急情况下该怎么做？" → ✗ 紧急不适用渐进

### 边界模糊
- "公司转型该多快？" → ✓ 组织变革节奏

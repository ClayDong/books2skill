# 渐进式改革路径设计框架

## R — 原文 (Reading)

> "共同富裕是分阶段推进的——不能犯急性子，不同阶段有不同标准。缩小差别不等于消灭差别——基尼系数0.3-0.4、区域/城乡差距控制在2倍以内是合理区间。"
>
> — 厉以宁等，总论

---

## I — 方法论骨架 (Interpretation)

改革路径设计需遵循三大原则：

**原则1: 分阶段推进**
- 不设终极目标，设阶段性目标
- 每阶段有明确的时间表和量化标准
- 阶段间有过渡期，避免断崖式切换

**原则2: 合理区间**
- 不追求绝对均等，设定"合理区间"
- 基尼系数：0.3-0.4（0.4为警戒线）
- 区域/城乡差距：控制在2倍以内
- 差距是激励创新的动力，完全消灭=失去活力

**原则3: 可调整**
- 保留政策调整空间
- 根据实际效果动态修正
- 设立反馈机制和评估周期

关键洞察：急性子改革导致资本外流+企业家信心崩溃+社会动荡。慢不是目的，稳是目的。

## A1 — 书中的应用 (Past Application)

### 案例 1: 中国改革开放的渐进性
- **问题**: 为什么中国改革成功了而苏联休克疗法失败？
- **方法论的使用**: 中国：渐进式（特区试点→推广）vs 苏联：激进式（休克疗法）
- **结论**: 渐进式改革允许试错和调整，激进式改革风险不可控
- **结果**: 中国40年持续增长，苏联经历十年衰退

### 案例 2: 共同富裕的阶段性目标
- **问题**: 共同富裕的时间表是什么？
- **方法论的使用**: 
  - 2020-2035：基本实现，中等收入群体翻番
  - 2035-2050：全面实现，橄榄型结构稳固
- **结论**: 不急于求成，分两步走

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 改革设计者——"这个改革该怎么推进？一步到位还是分步走？"
2. 政策评估者——"这个改革节奏合理吗？"
3. 组织变革者——"公司转型该多快？"
4. 个人决策——"职业转型该激进还是渐进？"

## E — 可执行步骤 (Execution)

1. **设定阶段性目标**
   - 完成标准: 至少分3个阶段，每阶段有量化目标
   - 判停条件: 每阶段不超过5-10年

2. **定义合理区间**
   - 完成标准: 明确"成功"的区间范围，而非单一目标值
   - 追问: "差距控制在多少以内是可接受的？"

3. **设计试点机制**
   - 完成标准: 先小范围试点，验证后推广
   - 判停条件: 试点期至少1-2年

4. **建立反馈调整机制**
   - 完成标准: 设立评估周期（如每年），明确调整触发条件

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 紧急危机——需快速反应，不能渐进
- 不可逆决策——如主权让渡、宪法修改等需慎重
- 已有成熟方案——不需要试错

### 作者在书中警告的失败模式

- 急性子改革导致社会动荡（ce03）
- 追求绝对均等导致失去活力

### 作者的盲点 / 时代局限

- "渐进"可能成为拖延的借口——何时该加速？
- 对"合理区间"的量化标准缺乏理论推导——0.3-0.4为何合理？

### 容易混淆的邻近方法论

- 休克疗法：激进改革，与渐进式对立
- 敏捷迭代：类似但用于产品开发，本框架用于政策改革

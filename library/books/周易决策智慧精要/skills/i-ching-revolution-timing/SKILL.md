# 变革时机决策 (Revolution Timing Decision)

## R — 原文 (Reading)

> 革,巳日乃孚,元亨利贞,悔亡。
>
> — 《革卦》

> 革,水火相息,二女同居,其志不相得,曰革。巳日乃孚,革而信之。文明以说,大亨以正。革而当,其悔乃亡。
>
> — 《革卦·彖传》

> 鼎,元吉,亨。
>
> — 《鼎卦》

> 革物者莫若鼎,故受之以鼎。
>
> — 《序卦传》

> 神农氏没,黄帝尧舜氏作,通其变,使民不倦,神而化之,使民宜之。易穷则变,变则通,通则久。
>
> — 《系辞下传》

---

## I — 方法论骨架 (Interpretation)

易经的变革哲学: 变革 (革) + 鼎新 (鼎) = 完整的转型过程。

### 革与鼎的关系

**革 (Gé) — 打破旧秩序**
- 性质: 革命性变化
- 核心: 打破既有的平衡
- 风险: 革的过程中不稳定
- 比喻: 水的沸腾,火的燃烧 — 革是"水火相息" (相互转化)

**鼎 (Dǐng) — 建立新秩序**
- 性质: 创新性建立
- 核心: 在革之后建立稳定的新结构
- 风险: 鼎立不住则新秩序崩溃
- 比喻: 鼎作为古代的炊具,代表"养人之器" — 鼎是"养人"的稳定结构

**顺序: 先革后鼎,不可颠倒**
- 革是破,鼎是立
- 不革则无以鼎新
- 革完不鼎则混乱持续
- 序卦: "革物者莫若鼎" — 革是为了鼎

### 革的时机四要素

> "巳日乃孚,元亨利贞,悔亡"

| 要素 | 含义 | 革的判断 |
|---|---|---|
| **巳日** | 时机成熟 | 变革的"天时"已到,人心思变 |
| **乃孚** | 众人信服 | 不是少数人的意志,是共识 |
| **元亨** | 大通顺 | 变革方向符合大势 |
| **利贞** | 合乎正道 | 变革不是为私利,是为大家 |
| **悔亡** | 悔恨消失 | 变革后不后悔 |

**四要素全满足才能革**,缺一不可:
- 缺"巳日" → 时机未到,强革必败
- 缺"乃孚" → 众人不支持,革难以推进
- 缺"元亨" → 方向不对,革完更糟
- 缺"利贞" → 不合正道,反受其害

### 鼎的"元吉"四要素

> "鼎,元吉,亨"

- **元**: 开始 — 鼎是新的开始
- **吉**: 吉祥 — 鼎立之后带来吉祥
- **亨**: 通达 — 鼎立之后各得其所
- **综合**: 鼎是"革"的成功标志 — 革完有吉有亨,才是真的鼎

### 革鼎决策的 5 阶段流程

#### 阶段 1: 革的必要性评估
- 不革会怎样? 5 年后回头看,会后悔没革吗?
- 如果答案是"会后悔" → 革的必要性成立
- 如果答案是"不会" → 不必革,优化即可

#### 阶段 2: 革的时机判断
```
巳日 (时机成熟): 
  - 行业/公司/个人 已到不得不变的临界点
  - 旧模式持续失灵
  - 新模式已隐约可见

乃孚 (众人信服):
  - 团队/家人/股东 都认为"该变了"
  - 不是你一个人的执念
  - 反对声音 < 支持声音
```

#### 阶段 3: 革的方向确定
- 革什么: 战略/组织/技术/产品/个人职业
- 不革什么: 核心价值观/核心竞争力/核心团队
- **保留 vs 革除**: 不能"全革",要保留根基

#### 阶段 4: 鼎的新秩序设计
- 革完的目标状态是什么?
- 谁是新秩序的核心?
- 资源/权力/责任如何重新分配?
- 失败的预案是什么?

#### 阶段 5: 革+鼎的执行
- 不要"全革" — 渐进革命更稳
- 准备"阵痛期" (1-3 个月混乱)
- 关键岗位要有"鼎"的人 (新秩序的支柱)
- 持续沟通 — 让大家看到"鼎"的希望

### 革的"勿做"清单

| 勿做 | 原因 |
|---|---|
| 时机未到强革 | "非所困而困" — 必败 |
| 革得太彻底 | 全革 = 无根之革,易翻车 |
| 革完不鼎 | 革完不立新,混乱持续 |
| 个人意志代众人 | 缺"乃孚",独夫之革 |
| 为私利而革 | 缺"利贞",反受其害 |
| 革完立即要求效果 | 鼎需要时间,不能急 |

## A1 — 书中的应用 (Past Application)

### 案例 1: 微软纳德拉转型 — 革+鼎的完整应用

- **问题**: 2014 年微软处于 PC 时代终结、移动时代崛起的转折点
- **革的必要性**: 5 年后回头看,不变必亡 (智能手机/云服务正在颠覆)
- **革的时机**:
  - 巳日: PC 销量持续下滑,新 CEO 接任
  - 乃孚: 内部共识 (鲍尔默下台即证明)
- **革的方向**:
  - 革: 打破"Windows 为中心"
  - 保留: Office/企业服务/技术研发能力
- **鼎的新秩序**:
  - 鼎 1: 移动优先 (iOS/Android 版 Office)
  - 鼎 2: 云优先 (Azure 业务)
  - 鼎 3: 重新成为企业服务领导者
- **结果**: 市值从 3000 亿到 3 万亿 (10 倍)
- **关键**: 纳德拉没有否定微软,而是在保留核心的同时"革"掉旧思维

### 案例 2: 苹果公司 2007 年 — 革手机行业

- **问题**: 2007 年手机行业被诺基亚/黑莓主导,功能机为主
- **革的必要性**: 5 年后手机必然智能化 (触屏/应用/网络)
- **革的时机**:
  - 巳日: 触屏技术成熟,3G 网络普及
  - 乃孚: 行业已经在讨论"智能手机"
- **革的方向**:
  - 革: 物理键盘 → 触屏 + 应用生态
  - 保留: 设计美学 + 用户体验 + 硬件整合
- **鼎的新秩序**:
  - 鼎 1: iPhone 重新定义手机
  - 鼎 2: App Store 生态
  - 鼎 3: 移动互联网时代
- **结果**: 苹果成为全球市值最高公司

### 案例 3: 文王演周易 — 文化层面的"革"

- **问题**: 商朝末年,占卜文化混乱,缺乏系统性
- **革的方向**: 把八卦重为六十四卦,建立系统
- **鼎的新秩序**: 周易成为后世占卜+哲学的根本
- **结果**: 影响中国文化 3000 年

### 反例: 王莽改革 — 革的失败

- **问题**: 西汉末年社会矛盾激化
- **革的时机问题**:
  - 缺"巳日": 民众仍认同汉朝
  - 缺"乃孚": 改革派 vs 保守派斗争激烈
  - 缺"利贞": 改革有私利嫌疑
- **革的方向问题**:
  - 全革: 全面否定汉朝制度
  - 没保留: 失去文化连续性
- **鼎的失败**: 改革配套措施不到位,基层执行混乱
- **结果**: 改革失败,新朝 15 年而亡

## A2 — 触发场景 (Trigger Scenarios)

### 场景 1: 创业公司转型
- 用户:"初创公司增长停滞,该不该转型"
- 应用: 革的时机四要素评估
- 输出: 转型方向 + 时机判断 + 鼎的方案

### 场景 2: 传统企业数字化
- 用户:"传统制造企业该不该全面数字化"
- 应用: 革的必要性 + 时机
- 输出: 渐进式革 vs 革命式革 的选择

### 场景 3: 个人职业革命
- 用户:"35 岁了,该不该彻底转行"
- 应用: 革的时机 + 个人"鼎" (新身份)
- 输出: 转行的可行性评估

### 场景 4: 团队重组
- 用户:"团队效率低,该不该重组"
- 应用: 革的范围 (局部 vs 全面)
- 输出: 重组方案

### 场景 5: 技术栈革命
- 用户:"该从 MySQL 迁到 TiDB 吗"
- 应用: 革的时机 (业务量) + 鼎 (新架构)
- 输出: 迁移计划

## E — 执行 (Execution)

### 步骤 1: 革的必要性评估 (1-2 天)

```
1. 不革会怎样? (5 年视角)
   - 继续不变,5 年后我会/公司会/项目会怎样?
   - 评估: 死/衰退/维持/不确定

2. 革完会怎样? (乐观/悲观 两种情景)
   - 乐观: 革完达到目标的可能性
   - 悲观: 革完失败的后果是否可承受

3. 决策:
   - 不革 = 死 → 必须革
   - 不革 = 衰退 → 强烈建议革
   - 不革 = 维持 → 暂缓,等更明确信号
   - 不革 = 不确定 → 收集更多信息
```

### 步骤 2: 革的时机四要素评估 (1 周)

```
巳日 (天时):
  - 外部环境已变,旧模式失灵 [是/否]
  - 新技术/新模式/新机会已出现 [是/否]
  - 评分: 0-10

乃孚 (人和):
  - 团队/家人/股东 已认识到"该变了" [是/否]
  - 反对声音 < 30% [是/否]
  - 评分: 0-10

元亨 (方向):
  - 革的方向符合大势 [是/否]
  - 5-10 年仍有价值 [是/否]
  - 评分: 0-10

利贞 (正道):
  - 革的目的不是私利 [是/否]
  - 经得起公开检验 [是/否]
  - 评分: 0-10

总分: ___ /40
- 32+ → 革的时机已到,启动
- 24-32 → 准备期,3 个月内启动
- 16-24 → 暂缓,补足薄弱要素
- <16 → 不革
```

### 步骤 3: 革的方向确定 (1-2 周)

```
1. 列出所有可能的方向 (不限数量)

2. 评估每个方向:
   - 必要性 (不变会死吗)
   - 可行性 (我有能力/资源吗)
   - 风险性 (失败后果可承受吗)
   - 长期价值 (5-10 年仍有价值吗)

3. 选定 1-2 个主要方向

4. 明确保留什么 (核心价值/能力/资源)
```

### 步骤 4: 鼎的新秩序设计 (2-4 周)

```
1. 革完的目标状态 (3 年视角):
   - 我/公司/项目 3 年后是什么样子?

2. 关键岗位/角色:
   - 谁是"鼎"的核心? (新秩序的支柱)
   - 谁会被淘汰? (旧秩序的代表)

3. 资源/权力/责任:
   - 如何重新分配?
   - 失败预案是什么?

4. 关键里程碑:
   - 3 个月/6 个月/12 个月/24 个月
   - 每个阶段的"鼎"标志是什么
```

### 步骤 5: 执行 + 监测 (持续)

```
第一阶段 (1-3 月): 革的阵痛期
  - 不要期待立竿见影
  - 监测: 团队是否稳定 / 业务是否滑坡
  - 调整: 必要时调整革的节奏

第二阶段 (3-12 月): 鼎的建立期
  - 新秩序的支柱是否就位
  - 旧的"包袱"是否卸下
  - 监测: 新指标是否改善

第三阶段 (12-24 月): 鼎的巩固期
  - 新秩序是否稳定
  - 团队是否进入新节奏
  - 复盘: 哪些革对了,哪些需要调整
```

## B — 边界 (Boundary)

### 不适用场景

- **渐进式优化**: 微调/迭代 — 用其他 skill
- **没有变革决心**: 本 skill 假设用户已决心要变
- **日常决策**: 吃什么/买什么 — 不需"革"

### 局限

- **易经的"革"偏重大变化**,对"小步快跑"的现代敏捷方法不完全适用
- **"巳日"判断主观**: 时机是否成熟有较强主观性
- **"鼎"需要时间**: 期望立竿见影会失望
- **"孚"难以量化**: 众人是否真心支持,难以准确评估

### 失效条件

- **环境剧变**: 战争/革命/疫情 — 易经"革"的框架失效
- **资源严重不足**: 革的代价无法承受
- **团队完全失能**: 已经崩溃的团队无法"革"和"鼎"

### 与现代变革管理的整合

| 易经概念 | 现代工具 | 整合方式 |
|---|---|---|
| 革的时机 | 战略转折点 / 颠覆式创新理论 | 用 Christensen 框架辅助判断 |
| 革的方向 | 商业模式画布 / V2MOM | 用结构化方法设计方向 |
| 鼎的新秩序 | Kotter 8 步法 / OKR | 用现代管理工具建立新秩序 |
| 乃孚 (共识) | 共创工作坊 / 调研 | 用集体智慧建立共识 |

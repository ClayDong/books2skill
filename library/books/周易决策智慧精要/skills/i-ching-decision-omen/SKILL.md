# 观几决策 (Omen-Based Decision)

## R — 原文 (Reading)

> 几者,动之微,吉之先见者也。君子见几而作,不俟终日。易曰:"介于石,不终日,贞吉。"介如石焉,宁用终日,断可识矣。君子知微知彰,知柔知刚,万夫之望。
>
> — 《系辞下传》

## I — 方法论骨架 (Interpretation)

易经决策哲学的**最高境界**:"见几而作,不俟终日" — 识别"几微"就行动,不等到第二天。

### "几"的概念

**几 (jī) 的定义**:
- 几 = 动之微 = 变化的最初征兆
- 几 = 吉之先见 = 吉祥 (或凶险) 到来之前的最早信号
- 几 = 看似不可见但确实存在的早期信号

**几的特点**:
- **微**: 微弱,不易察觉
- **先**: 先于明显变化
- **真**: 是真信号,不是噪声
- **动**: 预示即将到来的变化

**几 vs 噪声的区分**:

| 维度 | 几 (真信号) | 噪声 (假信号) |
|---|---|---|
| 可重复 | 是 (多次出现) | 否 (一次性) |
| 因果 | 与结果有因果关系 | 与结果无因果关系 |
| 时序 | 在结果之前 | 与结果同时/之后 |
| 强度 | 微弱但持续 | 强烈但短暂 |
| 多个维度 | 多维度同时出现 | 单维度偶发 |

### 几的 4 种类型

**1. 上升几 (正向早期信号)**
- 表现: 行业/公司/项目 出现新的积极苗头
- 案例: 智能手机出现初期 (2007-2010),iPhone 出货量增长
- 行动: 提前布局
- 风险: 误判为噪声

**2. 下降几 (负向早期信号)**
- 表现: 看似稳定的系统出现第一道裂缝
- 案例: 2008 年金融危机前的次贷危机苗头
- 行动: 提前撤离
- 风险: 过早撤退错过反弹

**3. 转折几 (方向变化的信号)**
- 表现: 趋势从上升转为下降,或反之
- 案例: 移动互联网向产业互联网转折
- 行动: 调整策略
- 风险: 误判方向

**4. 危机几 (重大风险的信号)**
- 表现: 系统性问题即将爆发
- 案例: 战争前夕的政治信号
- 行动: 立即应对
- 风险: 反应过度

### 见几而作 — 5 步决策法

#### 步骤 1: 观 (观察)
- 持续观察"小变化"
- 不被"大事件"分散注意力
- 建立"几"的记录系统 (笔记/数据库)

#### 步骤 2: 辨 (辨别)
- 用"几 vs 噪声"清单判断
- 多维度交叉验证
- 听取不同意见 (避免确认偏误)

#### 步骤 3: 决 (决策)
- 一旦确认"几"是真的,立即行动
- "不俟终日" — 不要拖到第二天
- 决策要有勇气,也要有预案

#### 步骤 4: 行 (执行)
- 小步试错 — 用小成本测试
- 持续监测 — 确认"几"在增强
- 准备撤退 — 如果"几"判断错误,能撤回

#### 步骤 5: 验 (验证)
- 1 周/1 月/1 季后回看
- 决策效果是否符合"几"的预期
- 学习: 哪类"几"判断准,哪类不准

### 几的"勿做"清单

| 勿做 | 后果 |
|---|---|
| 把噪声当几 | 误判,行动错误 |
| 把几当噪声 | 错过机会,后悔莫及 |
| 几出现后不动 | 几不等人,机会消失 |
| 几未确认就大动 | 误判风险大 |
| 单维度看几 | 容易被表象迷惑 |
| 不持续观察 | 几来时不知道 |

## A1 — 书中的应用 (Past Application)

### 案例 1: 诸葛亮草船借箭 — 几的精准识别

- **问题**: 三国时,周瑜要求诸葛亮 3 天造 10 万支箭
- **几的识别**:
  - 观察: 长江有大雾的规律 (几日观察记录)
  - 辨别: 大雾 + 曹军谨慎 + 视野不佳
  - 决策: 这是"进攻的时机" — 几
- **行动**:
  - 草船 20 只 + 军士 + 鼓声
  - 故意暴露,引曹军射箭
- **结果**: 3 天拿到 10 万+ 支箭
- **核心**: 诸葛亮通过长期观察,把"几" (大雾规律) 转化为行动

### 案例 2: 巴菲特 1969 年解散基金 — 下降几的识别

- **问题**: 1969 年美国股市仍处牛市
- **几的观察**:
  - 整体市场估值已达历史高位
  - 越来越难找到"便宜股票"
  - 投机氛围浓厚
- **几的判断**: 牛市末期 → 必有转折
- **行动**:
  - 主动解散基金 (急流勇退)
  - 把钱还给投资人
- **结果**:
  - 解散后市场又涨了 2 年 (1971)
  - 但 1973-74 大跌 (跌 50%+)
  - 巴菲特的客户躲过暴跌
- **核心**: 巴菲特识别了"市场已无便宜股票"这个几,提前撤离

### 案例 3: 微软错过移动 (几的失败案例)

- **问题**: 2007-2010 移动互联网崛起
- **几的失败识别**:
  - iPhone 出现 (2007) → 微软没有认真对待
  - Android 出现 (2008) → 微软继续推 Windows Phone
  - 应用生态兴起 (2010) → 微软仍坚守 Office/Windows
- **结果**:
  - 错过整个移动互联网时代
  - 鲍尔默下台
  - 2014 年微软才醒悟 (纳德拉接任)
- **教训**: 几出现后不动 = 必败

### 案例 4: 王阳明龙场悟道 — 几的精神捕捉

- **问题**: 王阳明被贬龙场,精神危机
- **几的捕捉**:
  - 观察: 在石棺中"格物致知"
  - 辨别: 心的本体是"良知"
  - 决策: 突破朱熹的"格物"框架
- **结果**: 龙场悟道,创立心学
- **核心**: 王阳明在困境中捕捉到"心的几",完成精神突破

## E — 执行 (Execution)

### 步骤 1: 持续观察 (基础工作)

```
建立"几"记录系统:
  - 每日/每周 记录 3 个微小变化
  - 分类: 行业/公司/个人/关系
  - 标注: 真信号 / 噪声 / 不确定
  - 每月复盘: 哪些"几"真的发生了,哪些没有
```

### 步骤 2: 几的辨别 (面对具体决策时)

```
问自己 5 个问题:
  1. 这个变化可重复吗? (几次出现)
  2. 这个变化有因果链吗? (原因→结果)
  3. 这个变化在结果之前吗? (时序)
  4. 这个变化多维度吗? (多个独立指标)
  5. 不同专家怎么看? (避免确认偏误)

评分:
  - 5 个"是" → 高度可信的"几"
  - 3-4 个"是" → 可信的"几"
  - 1-2 个"是" → 可能是噪声
  - 0 个"是" → 噪声
```

### 步骤 3: 决策矩阵

| 几的可信度 | 建议 |
|---|---|
| 高度可信 (5个是) | 立即行动,不俟终日 |
| 可信 (3-4个是) | 1 周内决策 |
| 可能是噪声 (1-2个是) | 持续观察,不行动 |
| 噪声 (0个是) | 忽略 |

### 步骤 4: 小步试错 (行动阶段)

```
投入规模:
  - 高度可信: 投入 30-50% 资源
  - 可信: 投入 10-20% 资源
  - 不确定: 投入 <5% 资源 (纯测试)

监测频率:
  - 高度可信: 每日监测
  - 可信: 每周监测
  - 不确定: 每月监测

撤退信号:
  - 几没有按预期发展
  - 出现了反向证据
  - 时机已过 (几的窗口关闭)
```

### 步骤 5: 验证 + 学习 (1-3 月后)

```
复盘:
  1. 几的判断准不准?
  2. 行动速度合不合适?
  3. 投入规模合不合适?
  4. 撤退信号是否有效?

学习:
  - 哪类"几"你最容易识别?
  - 哪类"几"你最容易误判?
  - 提升方法: 多读 + 多记录 + 多复盘
```

## B — 边界 (Boundary)

### 不适用场景

- **完全明朗的趋势**: 已经广为人知,不需要"几"的捕捉
- **紧急生死时刻**: 没时间观察"几",只能凭本能
- **一次性明确事件**: 如离婚/重大疾病,不需要"几"
- **缺乏领域知识**: 对领域不了解,无法识别"几"

### 局限

- **主观性强**: 几的判断有较强主观性
- **确认偏误**: 容易看到自己想看的"几"
- **数据有限**: 早期信号数据量小,难以统计验证
- **事后诸葛亮**: 很多"几"是事后才被认为是几

### 失效条件

- **领域完全陌生**: 没有基础知识,无法判断
- **情绪失控**: 焦虑/贪婪 会扭曲"几"的判断
- **数据稀缺**: 没有持续的数据/观察,无法建立"几"的判断
- **过度敏感**: 看到所有变化都当"几",导致频繁行动

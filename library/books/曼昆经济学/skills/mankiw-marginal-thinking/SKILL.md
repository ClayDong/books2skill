# 边际决策法

## R — 原文 (Reading)

> "理性人通常通过比较边际收益与边际成本来做决策。当且仅当边际收益大于边际成本时，理性决策者才会采取行动。理性人考虑边际量。"
>
> — 曼昆，第1章

---

## I — 方法论骨架 (Interpretation)

大多数决策不是"全做或全不做"，而是"多做一点还是少做一点"。

边际思维的核心：不看总量和平均量，只看"多一单位的额外收益（MB）"vs"多一单位的额外成本（MC）"。

决策规则：
- MB > MC → 做这一单位
- MB < MC → 不做这一单位
- MB = MC → 最优停止点

关键洞察：平均成本/收益会误导决策。航空公司起飞前以低于平均成本的价格卖票仍可获利——因为边际成本（多一个乘客的燃油和零食）远低于平均成本。

---

## A1 — 书中的应用 (Past Application)

### 案例 1: 航空公司起飞前定价
- **问题**: 航班即将起飞，还有空座，要不要降价卖票？
- **方法论的使用**: 平均成本（总成本÷总座位）可能很高，但边际成本（多一个乘客的燃油+零食）极低。只要票价 > 边际成本，卖票就增加利润。
- **结论**: 降价卖票是理性的——即使票价低于平均成本
- **结果**: 航空业普遍采用动态定价，起飞前降价

### 案例 2: 竞争企业的产量决策
- **问题**: 企业该生产多少产品？
- **方法论的使用**: 竞争市场中价格=边际收益。企业生产到 P=MC 为止。如果 P < AVC（平均可变成本），短期停产。
- **结论**: 不看总利润是否为正，看边际——P>MC 就继续生产，P<AVC 就停产

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 企业主在定价——"我的成本是100元/件，所以卖120元"（用了平均成本而非边际成本）
2. 个人在时间分配——"已经学了3小时，要不要再学1小时"（看边际收益是否递减）
3. 投资者在仓位管理——"已经赚了20%，要不要加仓"（看边际风险收益比）
4. 产品经理在功能优先级——"加这个功能的边际开发成本 vs 边际用户价值"

### 语言信号

- "平均每个成本是..."
- "已经投入了这么多"
- "要不要再加一点"
- "全做还是不做"
- "整体来看..."

### 与相邻 skill 的区分

- 与 `mankiw-opportunity-cost` 的区别: 机会成本问"放弃什么"，边际思维问"多一单位值不值"。前者是选什么，后者是选多少。
- 与 `mankiw-incentive-analysis` 的区别: 边际思维是决策者的自我分析，激励分析是预测他人对边际变化的反应。

## E — 可执行步骤 (Execution)

1. **识别决策的"单位"是什么**
   - 完成标准: 明确定义"多一单位"指什么（1件产品/1小时/1个功能/1个客户）

2. **估算边际收益（MB）**
   - 完成标准: 量化"多做一单位"带来的额外收益
   - 判停条件: 如果MB无法量化，用"高/中/低"分级

3. **估算边际成本（MC）**
   - 完成标准: 量化"多做一单位"带来的额外成本（含时间、精力、资源）
   - 注意: 不用平均成本！只算增量成本

4. **比较 MB 与 MC**
   - 完成标准: MB > MC → 做；MB < MC → 停；MB ≈ MC → 已到最优

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 二元决策——结婚/不结婚、辞职/不辞职等不可分割的选择，没有"多做一点"
- 边际变化不可行——某些生产只能整批进行（规模经济），无法按单位调整
- 非线性突变——临界点附近边际分析失效（如化学反应的相变点）

### 作者在书中警告的失败模式

- 用平均成本代替边际成本——导致过早停产或过度生产（ce02）
- 沉没成本谬误——把已投入的当作边际成本的一部分（ce07）

### 作者的盲点 / 时代局限

- "理性人"假设——行为经济学证明人有"现状偏差"，系统性地拒绝边际调整
- 边际收益递减假设——网络效应下可能存在边际收益递增，此时最优策略是"全力投入"

### 容易混淆的邻近方法论

- 平均分析：总成本÷总数量 ≠ 边际成本（多一单位的增量成本）
- 增量分析：接近但范围更大——增量可能包含多个单位，边际严格指"一单位"

# 供求与弹性分析

## R — 原文 (Reading)

> "供给与需求的力量共同决定了市场中每种物品的价格和销售量。均衡价格是使供给量等于需求量的价格。需求价格弹性衡量需求量对价格变化的反应程度。"
>
> — 曼昆，第4-5章

---

## I — 方法论骨架 (Interpretation)

分析任何市场现象的三步法：

**第一步：供求模型**
- 需求曲线（D）：价格越低，需求量越大（向下倾斜）
- 供给曲线（S）：价格越高，供给量越大（向上倾斜）
- 均衡：D=S的交点，决定价格和数量
- 关键区分："需求增加"（曲线右移）≠ "需求量增加"（沿曲线移动）

**第二步：弹性分析**
- 弹性 = 数量变化% / 价格变化%
- |E|>1 富有弹性：价格变化引起更大的数量变化
- |E|<1 缺乏弹性：价格变化引起更小的数量变化
- 弹性决定：涨价/降价对总收益的影响

**第三步：政策评估**
- 价格上限 < 均衡价 → 短缺
- 价格下限 > 均衡价 → 过剩
- 税收归宿由弹性决定：弹性小的一方承担更多税负

---

## A1 — 书中的应用 (Past Application)

### 案例 1: 汽油税的税收归宿
- **问题**: 对汽油征税，谁承担税负？
- **方法论的使用**: 短期汽油需求缺乏弹性（开车习惯不变）→消费者承担大部分。长期需求更富有弹性（换省油车、搬家）→生产者承担更多。
- **结论**: 税收归宿由弹性决定，不是由法律规定决定
- **结果**: 弹性分析准确预测了汽油税的分配效应

### 案例 2: 农产品价格支持
- **问题**: 政府设最低收购价保护农民
- **方法论的使用**: 农产品需求缺乏弹性→价格下限导致大量过剩→政府收购奶油湖
- **结论**: 价格支持扭曲市场信号，导致浪费
- **结果**: 欧美农业补贴长期存在但效率低下

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 企业主定价——"涨价10%会赚更多还是亏更多？"（取决于弹性）
2. 政策制定者——"设价格上限/下限会有什么后果？"
3. 投资者——"供给增加/需求下降对价格影响多大？"
4. 供应链管理——"原材料涨价对最终产品价格的影响？"

### 语言信号

- "涨价会怎样"
- "需求大不大"
- "价格敏感"
- "税收谁承担"
- "供过于求/供不应求"

### 与相邻 skill 的区分

- 与 `mankiw-incentive-analysis` 的区别: 激励分析关注"行为怎么变"，供求分析关注"价格和数量怎么变"。
- 与 `mankiw-welfare-market-structure` 的区别: 供求分析假设竞争市场，市场结构分析处理非竞争情况。

## E — 可执行步骤 (Execution)

1. **识别市场并画出供求图**
   - 完成标准: 确定需求曲线和供给曲线的大致位置和斜率

2. **判断弹性**
   - 完成标准: 评估需求/供给弹性（高/中/低）
   - 判断依据: 替代品可得性、必需品vs奢侈品、时间长短、市场定义范围

3. **分析变化的影响**
   - 完成标准: 识别是"曲线移动"还是"沿曲线移动"，预测新均衡
   - 判停条件: 如果是曲线移动，需判断移动方向和幅度

4. **评估政策/定价效果**
   - 完成标准: 
     - 定价：弹性大→降价增收；弹性小→涨价增收
     - 税收：弹性小的一方承担更多
     - 价格管制：上限→短缺，下限→过剩

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 垄断市场——垄断者是价格制定者，不是价格接受者，供求模型不直接适用
- 信息不对称市场——柠檬市场等需要额外分析框架
- 网络效应市场——需求曲线可能向上倾斜（越多人用越值钱）

### 作者在书中警告的失败模式

- 混淆"需求增加"与"需求量增加"——前者是曲线移动，后者是沿曲线移动（ce05）
- 价格管制导致更差结果——上限→短缺，下限→过剩（ce04）

### 作者的盲点 / 时代局限

- 假设市场趋于均衡——调整过程可能很长，存在多重均衡
- 静态分析——未考虑动态调整路径和时间滞后

### 容易混淆的邻近方法论

- 均衡分析 vs 博弈论：前者假设价格接受者，后者分析策略互动

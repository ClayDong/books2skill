# 曼昆经济学 — 压力测试 (阶段 4 产出)

## 测试方法

对每个 skill 设计测试 prompt，包含三类：
- **应调用**: 场景明确匹配 skill 的触发条件
- **不应调用 (诱饵)**: 场景看似相关但实际不匹配
- **边界模糊**: 场景在边界上，需要判断

---

## 1. mankiw-opportunity-cost

### 应调用
- "我在纠结要不要读研，读研花30万，工作能赚50万" → ✓ 应激活
- "自己开店不用付自己工资，成本更低" → ✓ 应激活

### 不应调用 (诱饵)
- "我已经在这个项目上投了3个月了，不能放弃" → ✗ 沉没成本谬误，不是机会成本
- "今天午饭吃什么" → ✗ 无重大替代选项

### 边界模糊
- "要不要跳槽到新公司？" → 取决于是否有明确的替代选项比较

---

## 2. mankiw-marginal-thinking

### 应调用
- "我的产品平均成本100元，要不要接80元的订单？" → ✓ 应激活（看边际成本）
- "已经学了3小时，要不要再学1小时？" → ✓ 应激活

### 不应调用 (诱饵)
- "要不要辞职创业？" → ✗ 二元决策，非边际
- "这个项目平均回报率15%" → ✗ 平均值描述，非决策

### 边界模糊
- "要不要再加一个功能？" → ✓ 如果能定义"一个功能的边际成本/收益"

---

## 3. mankiw-incentive-analysis

### 应调用
- "我们设了KPI但员工只做被考核的" → ✓ 应激活
- "租金管制为什么导致住房短缺？" → ✓ 应激活

### 不应调用 (诱饵)
- "我该怎么选？" → ✗ 个人决策，无他人行为预测
- "今天股市为什么跌？" → ✗ 短期波动，非激励分析

### 边界模糊
- "用户会不会刷积分？" → ✓ 激励分析的典型场景

---

## 4. mankiw-comparative-advantage

### 应调用
- "我什么都比员工做得好，为什么还要授权？" → ✓ 应激活
- "团队里谁该做什么任务？" → ✓ 应激活

### 不应调用 (诱饵)
- "我该学Python还是Java？" → ✗ 个人技能选择，非分工
- "这个供应商比那个好" → ✗ 绝对优势比较，非机会成本比较

### 边界模糊
- "要不要把IT外包？" → ✓ 如果分析包含"自己做IT的机会成本"

---

## 5. mankiw-supply-demand-elasticity

### 应调用
- "涨价10%对收入影响多大？" → ✓ 应激活
- "政府对汽油征税谁承担？" → ✓ 应激活

### 不应调用 (诱饵)
- "这家公司是垄断的" → ✗ 市场结构问题，不是供求分析
- "GDP增长多少" → ✗ 宏观议题

### 边界模糊
- "疫情后房价会涨吗？" → ✓ 供求分析适用，但需考虑政策因素

---

## 6. mankiw-welfare-market-structure

### 应调用
- "这个行业的竞争格局怎样？" → ✓ 应激活
- "垄断导致多少效率损失？" → ✓ 应激活

### 不应调用 (诱饵)
- "该不该对富人多征税？" → ✗ 分配公平问题，非市场效率
- "怎么定价？" → ✗ 定价是供求+弹性问题

### 边界模糊
- "平台垄断该不该反垄断？" → ✓ 市场结构+福利分析

---

## 7. mankiw-externality-policy

### 应调用
- "碳排放该用碳税还是碳交易？" → ✓ 应激活
- "邻居噪音怎么管？" → ✓ 应激活

### 不应调用 (诱饵)
- "这个产品质量不好" → ✗ 产品问题，非外部性
- "该不该补贴这个企业？" → ✗ 需先确认是否有正外部性

### 边界模糊
- "平台假货问题" → ✓ 负外部性（假货影响平台信誉）

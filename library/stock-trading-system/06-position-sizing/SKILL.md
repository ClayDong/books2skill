# 头寸规模层 — Position Sizing

## R — 原文 (Reading)

> "海龟头寸规模单位 =（账户资金 × 1%）/（ATR × 乘数）。其中ATR是20日真实波幅均值，乘数代表每单位波动的美元价值。这样无论市场波动大小，每个头寸的风险都恒定。"
>
> — 柯蒂斯·费思，《海龟交易法则》第6章（OCR原文，page 5230-5260）

> "R代表初始风险，等于入场价减去止损价。如果你冒1%的风险买入，R就是账户的1%。头寸规模应该基于R来计算，而不是固定股数。"
>
> — 范·K·撒普，《通向财务自由之路》第12章（OCR原文，page 4100-4130）

---

## I — 方法论骨架 (Interpretation)

头寸规模的核心原则是"风险恒定"——每笔交易的最大亏损固定为总资金的1%，与市场波动大小无关。

**关键公式（修正版）**：
```
股数 = (总资金 × 1%) / (2 × ATR)
```

为什么是 2×ATR 而非 1×ATR？因为止损距离 = 入场价 - 2×ATR（见 07-atr-stop-loss），每股风险 = 2×ATR。若用 1×ATR 做分母，实际风险会翻倍到2%，违反1%风险红线。

撒普的R乘数视角：R = 入场价 - 止损价 = 2×ATR，头寸 = 风险金额 / R，与海龟公式殊途同归。

持仓限制：
- 单一股票 ≤ 总资金 25%（集中度风险控制）
- 总仓位 ≤ 总资金 80%（预留20%流动性）
- 加仓后必须重新计算 ATR 和止损价

## A1 — 书中的应用 (Past Application)

### 案例 1: 海龟头寸标准化
- **问题**: 不同期货品种波动差异巨大，如何统一头寸规模
- **方法论的使用**: 用ATR标准化波动，每个头寸的ATR风险=账户1%，无论品种波动大小都买对应数量
- **结论**: 波动大的品种少买，波动小的品种多买，风险恒定
- **结果**: 海龟在多个品种同时持仓时，总风险可控，不会因单一品种暴雷而全军覆没

### 案例 2: 撒普R乘数验证
- **问题**: 固定股数买入时，不同止损距离的交易风险不一致
- **方法论的使用**: 用R=入场价-止损价统一风险度量，头寸=风险金额/R
- **结论**: R乘数法与海龟ATR法在止损距离=2×ATR时完全等价
- **结果**: 撒普的学生用R乘数法管理头寸，长期夏普比率优于固定股数法

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **前置校验：确认风险参数已就绪**
   - 检查 01-risk-control-baseline 是否已运行
   - 完成标准：总资金、单笔风险上限（1%）、最大持仓限制已确认
   - 判停条件：若未跑 01-risk-control-baseline，则提示用户先运行，跳到结束

2. **输入采集**
   - 总资金（元）
   - ATR（由05-breakout-entry输出的自适应ATR；若未跑05则默认ATR14日真实波幅均值，元）
   - 当前价 / 入场价（元）
   - 止损价（元，来自 07-atr-stop-loss 的ATR止损 或 13-mean-reversion的布林带止损；若未跑07/13则用入场价-2×ATR自行计算）
   - 已有持仓（股数 + 成本价，用于加仓场景）
   - 完成标准：以上5项参数全部获取
   - 判停条件：若缺总资金→⚠️待补充，要求用户补全总资金；若缺ATR→⚠️待补充，要求用户补全ATR（优先用05输出的自适应ATR，否则用14日ATR）；若缺入场价→⚠️待补充，要求用户补全入场价

3. **计算单笔风险金额**
   - 单笔风险金额 = 总资金 × 1%
   - 硬上限校验：单笔风险金额 ≤ 总资金 × 2%（硬上限，超过则报错）
   - 完成标准：风险金额已确认

4. **计算建议股数**
   - 每股风险 = 入场价 - 止损价（止损距离）
   - 计算方式（二选一，优先用传入的止损价）：
     - 方式A（ATR止损，来自05/07）：每股风险 = 2 × ATR，建议股数 = (总资金 × 1%) / (2 × ATR)
     - 方式B（布林带止损，来自13）：每股风险 = 入场价 - 止损价，建议股数 = (总资金 × 1%) / (入场价 - 止损价)
   - 1%风险红线校验：止损金额 = 每股风险 × 建议股数 ≤ 总资金 × 1%，若超限则缩减股数
   - 向下取整到100股的整数倍（A股最小交易单位）
   - 若计算股数<100股 → 触发"高价股小资金"处理流程：
     1. 计算硬买100股的实际风险：实际风险 = 100 × 每股风险，实际风险占比 = 实际风险 / 总资金
     2. 计算硬买100股的实际仓位：实际仓位 = 100 × 当前价 / 总资金
     3. 若实际风险占比 ≤ 2%（硬上限）且实际仓位 ≤ 25% → ⚠️允许买入100股但标注"风险偏高(实际X%)"
     4. 若实际风险占比 > 2%或实际仓位 > 25% → ❌不可建仓，建议：
        - a. 换低价股（同行业同逻辑但股价更低的标的）
        - b. 买ETF/基金替代（如行业ETF，间接持有）
        - c. 增加资金至最低可交易规模（最低资金 = 100 × 当前价 / 25%）
   - 完成标准：股数已计算并取整，1%风险红线校验通过，或已给出不可建仓建议

5. **持仓限制校验**
   - 单一股票持仓金额 = 建议股数 × 当前价
   - 校验1：单一股票持仓金额 ≤ 总资金 × 25%
     - 若超限 → 缩减股数至 25% 上限，输出 ⚠️"已达单一股票持仓上限"
   - 校验2：加仓后总仓位 ≤ 总资金 × 80%
     - 若超限 → 缩减加仓股数至 80% 上限，输出 ⚠️"已达总仓位上限"
   - 校验3：单一行业持仓金额 ≤ 总资金 × 20%
     - 若超限 → 缩减股数至20%上限，输出 ⚠️"已达单一行业集中度上限"
   - 完成标准：持仓限制校验通过或已缩减

6. **加仓场景处理**
   - 若已有持仓：加仓后重新计算 ATR（用最新14日数据）
   - 加仓后新止损价 = max(原止损价, 新入场价 - 2×新ATR)
   - 加仓后总风险 = (原持仓股数 + 加仓股数) × (新入场价 - 新止损价)
   - 校验：加仓后总风险 ≤ 总资金 × 2%（硬上限）
   - 完成标准：加仓后风险已重新计算并校验

7. **输出格式**
   ```
   ✅建议买入 | 股票:XXXX | 股数:NNN股 | 仓位占比:XX%
   风险金额:XXXX元(总资金1%) | 每股风险:XX元(2×ATR)
   加仓后总仓位:XX% | 加仓后总风险:XXXX元(XX%)
   下一步:调用07-atr-stop-loss设定止损价
   ```
   ```
   ⚠️持仓超限 | 原因:单一股票>25% / 总仓位>80%
   已缩减至:NNN股 | 实际仓位:XX%
   ```
   ```
   ❌不可建仓 | 原因:加仓后总风险>2%硬上限
   ```

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 判断要不要买（走 05-breakout-entry）
- 止损价计算（走 07-atr-stop-loss，本 skill 的止损价是输入而非输出）
- 出场判断（走 08-trend-exit）
- 融资融券/杠杆场景：禁止。走01-risk-control-baseline去杠杆后再调用本skill
- 期权/期货（乘数不同，公式需调整）

### 作者在书中警告的失败模式

- **公式误用**：海龟原公式用ATR×乘数（乘数=合约大小），A股无乘数，直接用2×ATR做每股风险。常见错误是分母只用1×ATR，导致实际风险翻倍到2%
- **固定股数陷阱**：撒普警告"用固定股数买入是最差的风险管理方式"，因为不同股票波动差异巨大
- **加仓不重算ATR**：加仓后市场波动可能已变化，必须用最新ATR重新计算

### 作者的盲点 / 时代局限

- 海龟公式设计于期货市场，合约乘数固定；A股无乘数但最小交易单位100股，需向下取整
- 撒普R乘数假设可精确控制止损，A股跌停时无法止损，实际风险可能超过R
- 25%单一股票上限是经验值，对低波动蓝筹股可适当放宽

### 容易混淆的邻近方法论

- "风险恒定"vs"仓位固定"：风险恒定是每笔亏损=1%资金，仓位固定是每次买相同股数，两者完全不同
- "R乘数"vs"凯利公式"：R乘数是固定风险比例，凯利公式根据胜率赔率动态调整，凯利更激进

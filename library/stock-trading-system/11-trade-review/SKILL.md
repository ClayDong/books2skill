# 交易复盘层 — 五维度归因引擎

## R — 原文 (Reading)

> "我每天记录交易日志，每周回顾。不是为了记住赚了多少，而是为了找出我的错误模式。同样的错误犯两次是不可原谅的。"
>
> — 杰西·利弗莫尔，《股票大作手回忆录》

> "华尔街没有新鲜事，因为人性从未改变。你今天犯的错，前人早就犯过，也早就付出过代价。"
>
> — 杰西·利弗莫尔，《股票大作手回忆录》

> "一个人从自己的错误中学习是聪明的，但从别人的错误中学习是更聪明的。在投机中，最大的敌人不是市场，而是你自己。"
>
> — 杰西·利弗莫尔，《股票大作手回忆录》

> 注：五维度复盘框架（技术/基本面/风控/心理/市场面）为现代整合，非利弗莫尔原文。利弗莫尔原文贡献在于"错误模式识别"和"同样的错误犯两次"理念。三类归因法（判断错误/执行错误/系统缺陷）为现代整合。

---

## I — 方法论骨架 (Interpretation)

利弗莫尔的复盘哲学：交易日志不是流水账，是"错误模式识别器"。同样的错误犯两次不可原谅——因为第一次是学费，第二次是浪费。

五维度复盘框架把一笔交易拆成五个独立维度审视，避免"只看结果不看过程"：

1. **技术面**：入场信号是否符合系统规则？出场时机是否及时？技术指标是否矛盾？
2. **基本面**：选股逻辑是否正确？买入后基本面是否发生变化？估值是否合理？
3. **风控面**：仓位是否合理（单笔≤1%（默认）/2%（硬上限）总资金）？止损是否执行？是否有过度集中？
4. **心理面**：情绪是否影响了决策？纪律是否遵守？是否存在冲动交易？
5. **市场面**：大盘环境是否配合？板块是否同步？市场情绪是否极端？

利弗莫尔归因法是核心区分工具：把错误分为三类——**判断错误**（分析方向错了）、**执行错误**（分析对了但没按计划做）、**系统缺陷**（按计划做了但系统本身有问题）。三类错误的修复路径完全不同。

## A1 — 书中的应用 (Past Application)

### 案例1: 利弗莫尔的交易日志复盘
- **问题**: 利弗莫尔在1907年恐慌中做空获利，但随后在反弹中过早回补，少赚了大量利润
- **方法论的使用**: 他在日志中记录：入场信号正确（判断✓），但出场过早（执行✗）。归因为"执行错误"——被短期反弹吓到，没有按系统规则持有
- **结论**: 执行错误的修复路径是"强化纪律"，不是"修改系统"
- **结果**: 后续交易中他设定了明确的出场规则（跌破关键位才平仓），不再被日内波动干扰

### 案例2: 利弗莫尔的"同样的错误犯两次"
- **问题**: 利弗莫尔多次因为"听信内幕消息"而亏损，但每次亏完又忍不住听下一个
- **方法论的使用**: 复盘日志帮他识别出这个重复模式——归因为"心理面：社会认同偏差+避免怀疑倾向"。这不是判断错误也不是系统缺陷，是心理执行错误
- **结论**: 心理层面的重复错误需要"硬规则"切断——他规定自己不再接受任何内幕消息
- **结果**: 切断信息源后，这类错误不再发生

### 案例3: 五维度复盘发现系统缺陷
- **问题**: 一个趋势跟踪系统回测表现优秀，但实盘3个月持续亏损
- **方法论的使用**: 五维度复盘——技术面✓（信号正确）、基本面✓（选股逻辑OK）、风控面✓（仓位合理）、心理面✓（纪律执行）、市场面✗（大盘处于震荡市，趋势跟踪系统在震荡市反复止损）
- **结论**: 归因为"系统缺陷"——系统缺少市场状态过滤器，在震荡市中失效
- **结果**: 加入ADX>25的趋势过滤器后，震荡市不再开仓，系统回撤改善

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **单笔交易复盘**：用户刚完成一笔交易（盈或亏），想知道"做对了什么/做错了什么"
2. **批量交易复盘**：用户给出最近5-10笔交易记录，想找"共同问题/重复模式"
3. **亏损归因**：用户连续亏损，想知道"到底是判断错了还是执行错了还是系统有问题"
4. **盈利归因**：用户赚钱了，想确认"是运气还是能力"——避免把运气当能力
5. **定期回顾**：用户做周/月度交易回顾，找模式、定改进方向

### 语言信号 (用户的话里出现这些就应激活)

- "帮我复盘一下这笔交易"
- "这笔为什么亏了"
- "我哪里做错了"
- "最近几笔交易有什么共同问题"
- "这笔赚了是运气还是能力"
- "同样的错误我又犯了"
- "帮我回顾一下这周的交易"

### 与相邻 skill 的区分

- 与 `09-emotion-control` 的区别：情绪控制是"事前/事中"的实时检测，复盘是"事后"的归因分析。交易前情绪不对→情绪控制；交易后要找问题→复盘
- 与 `10-system-optimization` 的区别：系统优化看"整体期望值"（统计级），复盘看"单笔/批量归因"（个案级）。优化系统→系统优化；找单笔问题→复盘
- 与 `12-mindset-calibration` 的区别：复盘输出"错误分类+改进措施"（可执行），心态校准输出"心境建议"（不输出交易指令）

### 下一步调用指引

- 归因为"判断错误" → 调用 `10-system-optimization` 检查入场/出场逻辑是否需要优化
- 归因为"执行错误" → 调用 `09-emotion-control` 检查是否有情绪干扰导致未按计划执行
- 归因为"系统缺陷" → 调用 `10-system-optimization` 做系统级调整
- 归因为"心理面重复模式" → 调用 `12-mindset-calibration` 做心态校准

## E — 可执行步骤 (Execution)

**输入**：交易记录（入场价/出场价/仓位/止损价/盈亏）+ 当时市场数据（大盘走势/板块表现）+ 心理状态记录

### 步骤1: 五维度逐项分析

对每笔交易按以下五个维度逐项打分（✓符合 / ✗不符合 / ⚠️部分符合）：

| 维度 | 检查项 | 判断标准 |
|------|--------|---------|
| 技术面 | ①入场信号是否符合系统规则？ ②出场时机是否及时？ ③技术指标是否矛盾？ | 信号与系统规则一致=✓；提前/延后出场=✗ |
| 基本面 | ①选股逻辑是否正确？ ②买入后基本面是否变化？ ③估值是否合理？ | 逻辑与结果一致=✓；买入后基本面恶化=✗ |
| 风控面 | ①仓位是否合理（单笔≤1%（默认）/2%（硬上限）总资金）？ ②止损是否执行？ ③是否过度集中？ | 仓位≤1%（默认）/2%（硬上限）=✓；止损未执行=✗ |
| 心理面 | ①情绪是否影响决策？ ②纪律是否遵守？ ③是否存在冲动交易？ | 按计划执行=✓；冲动交易=✗ |
| 市场面 | ①大盘环境是否配合？ ②板块是否同步？ ③市场情绪是否极端？ | 大盘趋势向上=✓；逆大盘操作=✗ |

- **完成标准**: 五个维度全部打分完毕，记录✗和⚠️的维度
- **判停条件**: 若交易记录不完整（缺入场价或止损价）→ 标⚠️待补充，要求用户补全数据

### 步骤2: 利弗莫尔归因分类

根据步骤1的打分结果，归入以下三类之一：

| 归因类型 | 判断条件 | 修复路径 |
|---------|---------|---------|
| 判断错误 | 技术面✗ 或 基本面✗ | 调整分析逻辑/入场条件（走系统优化） |
| 执行错误 | 技术面✓ 但 心理面✗ 或 风控面✗ | 强化纪律/情绪管理（走情绪控制） |
| 系统缺陷 | 五维度✓但持续亏损 | 调整系统参数（走系统优化） |

- **完成标准**: 明确归因类型（判断/执行/系统），不允许多重归因——必须找到主因
- **判停条件**: 若无法明确归因（多个维度都✗）→ 标⚠️待补充，建议增加更多交易样本

### 步骤3: 模式识别（批量复盘时）

若复盘≥3笔交易，做跨交易模式识别：

1. **重复错误检测**：同样的归因类型出现≥2次 → 标记为"重复模式"，高优先级修复
2. **盈利模式检测**：盈利交易的共同特征 → 提炼为"优势模式"，可强化
3. **亏损模式检测**：亏损交易的共同特征 → 提炼为"风险模式"，需规避

- **完成标准**: 输出重复错误列表 + 优势模式 + 风险模式

### 步骤4: 输出复盘报告

```
复盘报告:
═══════════════════════════════════
交易: {标的} {方向} {入场价→出场价} {盈亏}
═══════════════════════════════════
五维度分析:
  技术面: ✓/✗/⚠️ — {具体说明}
  基本面: ✓/✗/⚠️ — {具体说明}
  风控面: ✓/✗/⚠️ — {具体说明}
  心理面: ✓/✗/⚠️ — {具体说明}
  市场面: ✓/✗/⚠️ — {具体说明}

归因分类: {判断错误/执行错误/系统缺陷}
主因: {具体原因}

改进措施: ✅可执行 — {具体动作}
系统优化建议: {若有}
下一步: {调用哪个skill}
═══════════════════════════════════
```

- **完成标准**: 输出完整复盘报告，包含五维度分析、归因分类、改进措施
- **输出三类**: ✅可执行（归因明确+改进措施具体） / ❌不可执行（数据不足无法归因） / ⚠️待补充（需更多样本）

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **实时情绪检测**：用户在交易中情绪失控时，走 `09-emotion-control`（复盘是事后的）
- **系统整体评估**：用户在问"我的系统期望值多少"时，走 `10-system-optimization`（复盘看单笔/批量，系统优化看整体统计）
- **心态调适**：用户在问"怎么调整心态"时，走 `12-mindset-calibration`
- **选股分析**：用户在问"买什么股票"时，本 skill 不负责选股

### 禁忌红线

- ❌ **不做事前预测**：复盘是事后归因，不预测"下一笔会怎样"
- ❌ **不替代止损纪律**：复盘发现"止损设得不好"不等于"可以不设止损"
- ❌ **不鼓励"事后诸葛亮"**：不能用事后的信息评判当时的决策——只能用当时可知的信息
- ❌ **归因不允许多重主因**：必须找到唯一主因，多重归因等于没归因

### 作者在书中警告的失败模式

- 利弗莫尔警告：复盘最大的陷阱是"自我安慰式归因"——把亏损归因为"运气不好"而非"判断/执行错误"
- 利弗莫尔警告：同样的错误犯两次不可原谅——第一次是学费，第二次是浪费
- 利弗莫尔警告：盈利时也要复盘——把运气当能力，下次会加倍偿还

### 作者的盲点 / 时代局限

- 利弗莫尔的经验来自20世纪初的美股，没有量化工具，复盘靠手工记录——现代有交易软件辅助，但归因逻辑不变
- 利弗莫尔最终破产自杀，说明他的复盘方法虽好但执行纪律有时失效——复盘能力≠执行能力
- 五维度框架是现代整合的，利弗莫尔原文主要强调技术面和心理面

### 容易混淆的邻近方法论

- **"交易日记"**：不够。交易日记只是记录，复盘是"记录+归因+模式识别+改进"。没有归因的日记只是流水账
- **"事后分析"**：片面。复盘不只是分析亏损，盈利也要复盘（确认是能力还是运气）

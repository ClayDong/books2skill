# 股票交易全套 Skill 体系 — System Overview

> 基于 13 本交易经典跨书整合，覆盖趋势交易完整闭环 + 均值回归策略 + 价值投资风控 + 东方决策哲学。
> 核心原则：去故事、去感悟、全量化、强边界、A股适配。

## 设计原则

### 1. 去故事去感悟
- 只提取有明确方法论、可落地执行、能量化标准的内容
- 所有判断必须有明确数字阈值

### 2. 模块独立 + 流派唯一
- 核心 skill 不跨流派混合（本体系为趋势交易方向）
- 同流派逻辑自洽

### 3. 风控锚定（强制校验）
- **前置校验**：任何交易操作前先跑 `risk-control-baseline`
- **后置校验**：输出前校验仓位、单笔风险、账户回撤

### 4. A股适配（核心修复）
- T+1：买入当日无法止损，次日集合竞价处理
- 涨跌停：涨停不追单，跌停排队卖出
- 交易成本：佣金+印花税+滑点计入净期望值
- 板块轮动：单一行业集中度≤20%

## 关键修复（vs 上一版）

| 修复项 | 旧版问题 | 新版修复 |
|---|---|---|
| 头寸公式 | 公式数学错误，实际风险2倍 | 股数=资金×1%/(2×ATR)，明确每股风险 |
| 加仓止损 | 止损移到最新加仓价下方 | max(原止损, 新入场价-2×ATR) |
| A股涨停 | 未处理涨停买不到 | 涨停不追单+次日开盘处理 |
| 市场过滤 | 只用20日线太滞后 | 增加60日线+板块强度 |
| 回撤保护 | 无账户级熔断 | 15%减仓/20%清仓 |
| 黑天鹅熔断 | 无极端行情保护 | 单日跌≥5%标记/≥8%减仓/≥10%清仓冷静3日 |
| 板块集中 | 无限制 | 单一行业≤20% |
| 交易成本 | 未计入 | 佣金+印花税+滑点计入净期望值 |
| 入场参数 | 20/55日太迟钝 | 10/20日突破更适配A股 |
| 出场规则 | 只用20日线 | 20日低点+浮盈>30%回撤15%减仓/浮盈>50%回撤20%清仓（5年回测优化） |
| 均值回归 | 无震荡市策略 | 13-mean-reversion: RSI<30+布林带下轨+量缩确认，与05通过市场状态互斥 |
| ROE阈值 | 统一≥12%误杀金融/车企 | 按行业分类：银行≥10%/保险≥10%/证券≥8%/新能源车≥10% |
| CANSLIM | EPS≥25%仅6%符合 | 按行业分类：成长股≥15%/白马股≥10%/周期金融不适用 |
| PE区间 | 统一10-25倍覆盖不足 | 按行业分类：金融4-8/消费15-25/医药25-50/科技25-50/周期PB1-3 |
| test-prompts | 4处P0错误 | 全部修正 |

## 质量红线

1. 每个 skill 有完整 R/I/A1/A2/E/B 六段
2. E 段含"输入→执行逻辑→输出"结构，全量化
3. B 段含"禁忌红线"
4. 前置校验：交易操作 skill 必须先跑 01
5. 后置校验：输出前校验仓位/单笔风险/回撤
6. test-prompts.json 的 should_not_trigger 容错为 0
7. 输出三类：可执行/不可执行/待补充

| 07 | 01 | 风险参数 | 07步骤1校验01已运行 |
| 08 | 01 | 回撤校验 | 08步骤3委托01黑天鹅熔断+账户级回撤校验 |

**契约变更流程**：
1. 先改本表的契约要求列
2. 再改调用方SKILL.md（确保传递参数）
3. 再改被调用方SKILL.md（确保接受参数）
4. 运行 `python validate_system.py --check contract` 验证
5. 0错误才能提交

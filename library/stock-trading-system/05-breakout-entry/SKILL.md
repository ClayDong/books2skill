# 突破入场层 — Breakout Entry

## R — 原文 (Reading)

> "系统1：以20日突破为基础的短期系统。系统2：以55日突破为基础的长期系统。突破是指价格超越了过去一定时期内的最高点或最低点。海龟们总是在突破发生时立即入市交易。"

> — 柯蒂斯·费思，《海龟交易法则》第4章（OCR原文，page 6614-6619）

> "关键点就是那些经历长期横盘后，价格突破整数关口或历史阻力位的时刻。在关键点买入，胜率最高。"

> — 埃德温·勒菲弗（记录利弗莫尔），《股票大作手回忆录》

> "买入要点之一：突破时必须要有成交量的配合，没有量的突破是假突破。突破前要有蓄势过程。"

> — 陈江挺，《炒股的智慧》

---

## I — 方法论骨架 (Interpretation)

突破入场的核心逻辑是"趋势启动确认"——价格突破关键阻力位，说明多方力量压倒空方，趋势可能延续。

三层确认体系：
1. **价格突破**：收盘价突破 N 日最高价（海龟用20/55日，A股节奏更快，修正为10/20日）
2. **成交量确认**：突破当日成交量 ≥ 5日均量×1.5（陈江挺要求量价配合）
3. **涨停过滤**：A股涨停板封死时不追单（买不进+次日可能高开低走），次日集合竞价再判断

利弗莫尔的"试探性建仓"原则：首仓只买计划仓位的 1/5，确认趋势后再加仓。

## A1 — 书中的应用 (Past Application)

### 案例 1: 海龟咖啡糖突破
- **问题**: 1983年咖啡糖期货长期横盘后突破，海龟是否入场
- **方法论的使用**: 系统1触发——价格突破20日高点，成交量放大，按头寸单位买入
- **结论**: 突破有效，趋势延续，海龟获利
- **结果**: 咖啡糖从2.5美分涨到6美分，海龟系统捕获了完整趋势

### 案例 2: 利弗莫尔伯利恒钢铁
- **问题**: 1915年伯利恒钢铁在50美元关口反复试探，何时入场
- **方法论的使用**: 等待价格突破整数关口50美元确认后，试探性建仓（首仓1/5）
- **结论**: 关键点突破确认后入场，胜率最高
- **结果**: 利弗莫尔在伯利恒钢铁上大赚，账户从0恢复到14万美元

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户持有某只股票的自选股，看到它今天大涨突破前高，问"能不能买"
2. 用户看到某只股票突破20日新高，问"这个突破是真的还是假的"
3. 用户问"突破后追进去，涨停了买不进怎么办"

### 语言信号 (用户的话里出现这些就应激活)

- "突破""创新高""突破阻力位""要不要追"
- "这个突破是真的吗""假突破怎么办"
- "涨停了买不进""明天还能买吗"

### 与相邻 skill 的区分

- 与 `06-position-sizing` 的区别：本 skill 判断"要不要买"，06 判断"买多少"
- 与 `07-atr-stop-loss` 的区别：本 skill 是入场判断，07 是入场后的止损管理
- 与 `08-trend-exit` 的区别：本 skill 解决"进场"，08 解决"出场"

### 下一步调用指引

- ✅ 入场信号 → 调用 `06-position-sizing` 计算具体股数
- ⚠️ 涨停待次日 → 次日集合竞价（9:25）处理
- ❌ 震荡市不适用 → 走 `13-mean-reversion`（大盘波动率分位<40%）
- ❌ 不入场 → 等待或重新选股（调用 `02-stock-screening`）

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **前置校验：确认风险参数已就绪**
   - 检查 01-risk-control-baseline 是否已运行
   - 完成标准：总资金、单笔风险上限（1%）、最大持仓限制已确认
   - 判停条件：若未跑 01-risk-control-baseline，则提示用户先运行，跳到结束
   - 大盘环境校验：指数>60日线 → ✅可入场；指数<60日线 → ❌空仓等待
   - 市场状态校验（与13-mean-reversion互斥，使用相同的大盘波动率分位指标 + ADX辅助）：
     - 大盘20日波动率分位 < 40% 且 ADX<15 → 极端无趋势 → ❌不适用本skill，提示用户走 13-mean-reversion
     - 大盘20日波动率分位 > 60% 且 ADX>25 → 趋势市 → ✅适用本skill，继续
     - 40% ≤ 分位 ≤ 60% 或 ADX在15-25之间 → 中性区间 → ⚠️谨慎使用，可继续但建议降低仓位至首仓15%
     - ADX（平均定向指数）辅助判据：ADX>25确认趋势存在，ADX<15确认极端无趋势（硬过滤），15-25为模糊区（软警告，允许入场）
     - 回测验证：ADX<15硬过滤（5年10股回测，年化5.57%、盈亏比1.63、胜率51.8%）；ADX<20硬过滤过严导致年化降至4.05%，故阈值定为15
   - 若已通过 02-stock-screening 选股，则大盘60日线过滤已在 02 完成，无需重复；但市场状态校验（波动率分位+ADX）仍需执行，因02未做此校验

2. **输入采集**
   - 股票代码
   - 当前价（收盘价或实时价）
   - 10日最高价、20日最高价
   - 20日均线价（5年回测验证新增：趋势过滤用）
   - 当日成交量、5日均量
   - 板块类型（主板/创业板/科创板）
   - 是否涨停（涨停=涨幅≥板块涨跌停限制×99%且封板）
   - 完成标准：以上8项参数全部获取
   - 判停条件：若缺价格数据→⚠️待补充，要求用户补全当前价/10日最高价/20日最高价/20日均线价；若缺成交量数据→⚠️待补充，要求用户补全当日成交量/5日均量

3. **突破确认**
   - 系统1（短周期）：收盘价 > 10日最高价 → 短期突破信号
   - 系统2（长周期）：收盘价 > 20日最高价 → 强趋势突破信号
   - 两个系统都触发时优先级最高
   - 完成标准：判定是否突破及突破强度

3.5 **20日线趋势过滤**（5年回测验证新增：减少逆势入场）
   - 输入：20日均线价（近20个交易日收盘价移动平均）
   - 若收盘价 ≥ 20日均线 → ✅趋势向上，继续
   - 若收盘价 < 20日均线 → ❌不入场，输出原因"价格在20日线下方，逆势突破风险大"
   - 完成标准：20日线趋势过滤判定完成

4. **涨停过滤**
   - 若涨停且封板（封单量>成交量×3，全天未开板）→ 输出 ⚠️涨停待次日，跳到步骤7
   - 若涨停但未封死（封单量≤成交量×3）→ 视为可成交，继续
   - 若开盘涨停后开板回落（开盘涨幅≥板块涨跌停限制×99% 且 盘中跌破开盘价）→ ❌不入场，标记为"假突破高风险日"，当日不入场，观察次日走势（原因：开盘涨停后回落说明抛压巨大，追入大概率被套）
   - 若未涨停 → 继续
   - 完成标准：涨停状态判定完成

5. **成交量确认 + 北向资金辅助**
   - 突破当日成交量 ≥ 5日均量 × 1.5 → ✅ 量价配合
   - 突破当日成交量 < 5日均量 × 1.5 → ❌ 缩量突破，输出不入场+原因"量价背离"
   - 北向资金辅助确认（A股特色，可选）：当日北向资金净流入>0 → 信号加分；净流出>10亿 → 信号减分，标记"北向资金流出，谨慎"
   - 完成标准：成交量有效性 + 北向资金辅助判定完成

6. **T+1 处理与入场信号输出**
   - 若当日可买入（非涨停）→ 输出 ✅入场信号
     - 入场价 = 当前价（或次日开盘价）
     - 首仓比例 = 计划仓位 × 20%（利弗莫尔试探性建仓原则，《股票大作手回忆录》第5章"试探性建仓"：先买20%试探方向，确认正确后再加仓）
     - 下一步：调用 06-position-sizing 计算具体股数
   - 若涨停待次日 → 输出 ⚠️涨停待次日
     - 次日集合竞价（9:25）判断：若高开≤3%且未涨停 → 可入场
     - 若高开>3% → 放弃（追高风险过大）
   - 完成标准：输出明确的入场/不入场/待次日信号

7. **输出格式**
   ```
   ✅入场信号 | 股票:XXXX | 入场价:XX元 | 首仓比例:20% | 突破类型:10日+20日 | 量比:1.8x | ATR(14日):X.XX元
   ⚠️涨停待次日 | 股票:XXXX | 次日策略:集合竞价判断 | 高开阈值:≤3%
   ❌不入场 | 股票:XXXX | 原因:缩量突破/未突破/涨停封死/假突破高风险日
   ```

8. **ATR 参数自适应**（波动率环境适配）
   - 常规市场：ATR(14日)
   - 高波动市场（当日 ATR > 20日平均 ATR 的 1.5 倍）：改用 ATR(10日)，更快反映波动变化
   - 低波动市场（当日 ATR < 20日平均 ATR 的 0.7 倍）：改用 ATR(20日)，过滤噪音
   - 用途：本 skill 输出的波动率参数供 06-position-sizing 计算股数和 07-atr-stop-loss 计算止损时使用

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 基本面选股阶段（用估值模型而非技术突破）
- 定投/网格交易（不依赖突破信号）
- 已持仓股票的加仓判断（走 06-position-sizing）
- ST/*ST股票（涨跌停5%，突破信号失真）
- 上市首日及次新股（无历史数据，10/20日高点无意义）

### 作者在书中警告的失败模式

- **假突破陷阱**：价格突破后迅速回落，陈江挺警告"没有量的突破是假突破"
- **追涨杀跌**：利弗莫尔警告"不要在突破后追高太多，入场价偏离突破价>5%就放弃"
- **涨停板陷阱**：A股涨停封死后次日可能高开低走，追入即套

### 作者的盲点 / 时代局限

- 海龟系统设计于1980年代美国期货市场，T+0、无涨跌停限制，直接搬到A股会水土不服
- 利弗莫尔时代无程序化交易，突破假信号比现在少
- 10/20日参数需根据A股板块特性微调（大盘股可放宽至15/30日）

### 容易混淆的邻近方法论

- "突破"vs"均线金叉"：突破看价格创新高，均线金叉看短期均线上穿长期均线，两者可同时出现但不等价
- "试探性建仓"vs"分批建仓"：试探是先小仓验证方向，分批是按计划分次买入

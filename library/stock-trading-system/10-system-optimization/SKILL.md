# 系统优化层 — R乘数与期望值引擎

## R — 原文 (Reading)

> "R是你在每笔交易中愿意承担的初始风险。如果你以10元买入，止损设在9元，那么R=1元。如果你最终以12元卖出，这笔交易就是+2R的收益。系统的质量不在于胜率，而在于期望值。"
>
> — 范·K·撒普，《通向财务自由之路》

> "反过来想，总是反过来想。如果你知道会死在哪里，就永远别去那个地方。"
>
> — 查理·芒格，《穷查理宝典》

---

## I — 方法论骨架 (Interpretation)

撒普的R乘数系统是交易系统评价的底层语言：把每笔交易的盈亏标准化为"风险的倍数"（R），而非绝对金额。这样不同仓位、不同标的的交易才可比较。

期望值公式是核心：**期望值 = 胜率 × 平均盈利R - 败率 × 平均亏损R**。期望值>0且>交易成本，系统才可交易。

芒格的四个思维工具补强了系统优化：
1. **逆向思维**：先问"这个系统怎么会亏钱"，再避免——而不是只想着怎么赚钱
2. **检查清单**：投资前逐项核对，防止遗漏关键风险因子
3. **多元思维模型**：用跨学科模型（物理学的临界质量、生物学的进化、心理学的偏差）分析标的，避免单一视角盲区
4. **lollapalooza效应**：多个因素同向叠加时会产生极端市场状态（如杠杆+从众+流动性枯竭=崩盘），系统设计必须考虑极端情况

本 skill 的关键修复：把交易成本（佣金万2.5+印花税千0.5+滑点千1）显式计入期望值计算，避免"纸面盈利实际亏损"。

## A1 — 书中的应用 (Past Application)

### 案例1: 撒普的R乘数系统评价
- **问题**: 一个交易者胜率60%，但总在亏钱，不知道为什么
- **方法论的使用**: 撒普让他用R乘数重新计算——胜率60%但平均盈利只有0.5R，平均亏损却是1.5R。期望值 = 0.6×0.5 - 0.4×1.5 = 0.3 - 0.6 = -0.3R
- **结论**: 胜率高不等于赚钱，盈亏比才是关键。这个系统期望值为负，不可交易
- **结果**: 交易者调整止损策略，把盈亏比从0.33提升到2.0，期望值转正

### 案例2: 芒格的逆向思维应用
- **问题**: 设计一个趋势跟踪系统，怎么确保它不会大亏
- **方法论的使用**: 芒格的逆向思维——先列出"趋势跟踪系统怎么亏钱"：①震荡市反复止损 ②趋势反转时回吐利润 ③黑天鹅跳空止损失效
- **结论**: 针对①加震荡过滤器，针对②用移动止损，针对③限制单笔风险≤1%（默认）/2%（硬上限）总资金
- **结果**: 系统在回测中最大回撤从35%降到18%

### 案例3: 芒格的lollapalooza效应识别
- **问题**: 2008年金融危机前，为什么那么多"聪明"的系统都失效了
- **方法论的使用**: 芒格的lollapalooza效应——杠杆+次贷+评级机构失职+从众+流动性枯竭，多个因素同向叠加产生极端状态，正常概率分布失效
- **结论**: 系统设计必须考虑"多因素叠加"的极端情况，不能只看正态分布下的回测
- **结果**: 在系统中加入"lollapalooza检测"——当3个以上风险因子同向时，自动降仓至30%以下

1. **评估交易系统好坏**：用户给出一段历史交易记录，问"我的系统到底赚不赚钱""能不能靠这个吃饭"

4. **系统失效诊断**：用户问"以前赚钱的系统最近不灵了，怎么回事"
5. **新系统上线前检查**：用户设计了一个新策略，想在上实盘前做系统级检查

### 语言信号 (用户的话里出现这些就应激活)

- "我的交易系统期望值是多少"
- "胜率多少才能盈利"
- "盈亏比多少才合理"

### 与相邻 skill 的区分

- 与 `09-emotion-control` 的区别：本 skill 评估的是"系统的数学质量"（期望值/R乘数），情绪控制评估的是"人的心理状态"。先调情绪控制确认状态OK，再调本 skill 做系统分析
- 与 `11-trade-review` 的区别：本 skill 是"系统级"的统计评价（看整体期望值），复盘是"单笔级"的归因分析（看具体哪笔为什么亏）。优化系统→本 skill；找单笔问题→复盘
- 与 `12-mindset-calibration` 的区别：本 skill 输出量化判断（可交易/不可交易），心态校准输出心境建议（不输出交易指令）

### 下一步调用指引

- 期望值>0且>交易成本 → ✅可执行：系统可交易，调用 `11-trade-review` 持续复盘优化
- 期望值≤0 → ❌不可执行：系统不可交易，需要调整入场/止损/出场逻辑后重新评估
- 期望值>0但接近交易成本 → ⚠️待补充：系统边际，建议增加样本量（至少30笔）再判断

## E — 可执行步骤 (Execution)

**输入**：历史交易记录（每笔含：入场价、止损价、出场价、方向、盈亏金额、总资金）

### 步骤1: 计算每笔交易的R乘数

对每笔交易计算R值和R乘数：

```
R = |入场价 - 止损价|    （单笔风险金额 = R × 仓位股数）
R乘数 = (出场价 - 入场价) / R    （做多）
R乘数 = (入场价 - 出场价) / R    （做空）
```

- **完成标准**: 每笔交易都有一个R乘数值（正=盈利，负=亏损）
- **判停条件**: 若交易记录<10笔 → 标⚠️待补充，样本量不足，建议至少积累30笔再评估

### 步骤2: 计算系统期望值

```
胜率 = 盈利笔数 / 总笔数
败率 = 1 - 胜率
平均盈利R = 所有盈利交易的R乘数之和 / 盈利笔数
平均亏损R = 所有亏损交易的R乘数之和 / 亏损笔数（取绝对值）
期望值(R) = 胜率 × 平均盈利R - 败率 × 平均亏损R
```

- **完成标准**: 输出胜率、平均盈利R、平均亏损R、盈亏比（平均盈利R/平均亏损R）、期望值(R)

### 步骤3: 扣除交易成本

交易成本参数（固定）：
- 佣金：万2.5（双边，即买入+卖出各万2.5）
- 印花税：千0.5（仅卖出）
- 滑点：千1（双边，即买入+卖出各千1）

```
买入单边 = 佣金0.025% + 滑点0.1% = 0.125%（千1.25）
卖出单边 = 佣金0.025% + 印花税0.05% + 滑点0.1% = 0.175%（千1.75）
双边合计 = 0.3%（千3.0）
```

将总成本率转换为R：
```
成本R = 总成本率 × 入场价 / R
净期望值 = 期望值(R) - 成本R
```

- **完成标准**: 输出毛期望值(R)、成本R、净期望值(R)

### 步骤4: 系统评价与优化建议

根据净期望值分级：

| 分级 | 条件 | 评价 | 建议 |
|------|------|------|------|
| ✅可执行 | 净期望值 > 0.3R | 系统优秀 | 可交易，持续复盘优化 |
| ⚠️待补充 | 0 < 净期望值 ≤ 0.3R | 系统边际 | 增加样本量验证，优化盈亏比 |
| ❌不可执行 | 净期望值 ≤ 0 | 系统不可交易 | 需调整入场/止损/出场逻辑 |

芒格逆向思维检查清单（逐项核对）：
1. ☐ 震荡市是否反复止损？（若是→加趋势过滤器）
2. ☐ 趋势反转时是否回吐过多利润？（若是→收紧移动止损）
3. ☐ 黑天鹅跳空时止损是否失效？（若是→限制单笔风险≤1%（默认）/2%（硬上限）总资金）
4. ☐ 是否存在lollapalooza风险？（3个以上风险因子同向时→自动降仓至30%）

- **完成标准**: 输出系统评价（✅/❌/⚠️）+ 逆向思维检查结果 + 优化建议
- **输出格式**:
  ```
  系统评价: ✅可执行 / ❌不可执行 / ⚠️待补充
  胜率: {%}
  盈亏比: {平均盈利R : 平均亏损R = X:1}
  毛期望值: {X}R
  交易成本: {X}R
  净期望值: {X}R
  逆向检查: {命中项}
  优化建议: {具体建议}
  下一步: {调用哪个skill}
  ```

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **单笔交易的情绪评估**：用户在纠结"这笔亏了怎么办"时，走 `09-emotion-control`
- **单笔交易的归因分析**：用户在问"这笔为什么亏"时，走 `11-trade-review`
- **心态调适**：用户在问"怎么调整心态"时，走 `12-mindset-calibration`
- **选股逻辑**：用户在问"买什么股票"时，本 skill 不负责选股

### 禁忌红线

- ❌ **样本量<10笔不做最终评价**：统计意义不足，只能标⚠️待补充
- ❌ **不忽略交易成本**：佣金+印花税+滑点必须计入，纸面盈利≠实际盈利
- ❌ **不保证未来收益**：历史期望值>0不等于未来一定赚钱，市场结构会变化
- ❌ **不替代情绪控制**：系统再好，情绪失控时执行也会变形

### 作者在书中警告的失败模式

- 撒普警告：过度优化（curve-fitting）——在历史数据上调出完美参数，但实盘失效。建议留30%样本做样本外验证
- 芒格警告：单一思维模型的陷阱——只用技术分析不看基本面，只用基本面不看宏观，都会盲人摸象
- 芒格警告：lollapalooza效应被低估——正常回测无法模拟多因素叠加的极端情况

### 作者的盲点 / 时代局限

- 撒普的R乘数假设止损能被执行，但A股有涨跌停板，极端行情下止损可能跳空失效
- 芒格的多元思维模型是定性框架，缺乏量化标准——本 skill 用"3个以上风险因子同向→降仓30%"做了量化补强
- 交易成本参数基于A股当前费率，若费率调整需更新

### 容易混淆的邻近方法论

- **"胜率越高越好"**：错误。胜率60%但盈亏比0.3的系统，期望值为负。胜率和盈亏比是跷跷板
- **"回测好就能赚钱"**：不够。回测有过度优化风险，且无法模拟lollapalooza效应

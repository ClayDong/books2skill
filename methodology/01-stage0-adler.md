# 阶段 0 — 整书理解 (Adler 分析阅读)

## 目标

在动手拆书之前,先**真正读懂这本书**。没有这一步,拆出来的 skill 只是一堆金句集,会带上作者的盲点却不自知。

产出: `books/<slug>/BOOK_OVERVIEW.md` (按 `templates/BOOK_OVERVIEW.md.template` 填充)。

## 阶段 -1 — 快速扫描 (多书模式前置) ★

**仅多书模式执行。** 单本模式可跳过直接进入阶段 0。

在精读之前,先花 5% 的时间判断这本书值不值得精读、在知识库中扮演什么角色。

### 扫描内容 (只读这些)

- 目录 (全书骨架)
- 序言/前言 (作者自述主旨)
- 结语/后记 (作者总结)
- 每章首段 + 末段 (章节核心)
- 索引/术语表 (高频概念)

### 扫描产出 (SCAN_NOTES.md)

```yaml
book: {{书名}}
scan_time: {{DATE}}
estimated_value: high / medium / low
estimated_skill_count: {{N}} (粗估)
role_in_library: {{基础理论 / 实操方法 / 哲学视角 / 政策背景 / 案例素材}}
priority: high / medium / low
key_topics: [{{主题1}}, {{主题2}}]
potential_conflicts_with: [{{已有书名}}]  # 可能与哪些书有冲突
scan_summary: |
  (3-5 行, 这本书讲什么, 值不值得精读, 在知识库中的定位)
```

### 多书排序原则

扫描完所有书后,按以下原则排序精读:
1. **基础理论优先**: 其他书依赖的概念来源先读
2. **方法论密集优先**: 同等价值下,能产出更多 skill 的先读
3. **冲突点优先**: 与已读书有冲突的,早读早记录
4. **厚度倒序**: 薄的先读,快速验证流水线

## 阶段 0 — 分层精读策略

### 大文件分块阅读规则

- **分块单位**: 优先按章节分块 (保持语义完整), 不按固定 token 数
- **分块大小**: 单块 ≤ 30000 token (约 15-20 页中文)
- **块间衔接**: 每块开头重读上一块末尾 2 段 (保持上下文)
- **笔记同步**: 每块读完后立即写 3-5 行摘要到 `books/<slug>/reading_notes.md`, 不依赖记忆

### 三层阅读法 (替代"读一遍")

| 层次 | 目标 | 速度 | 产出 |
|---|---|---|---|
| L1 扫描 | 判断值不值得精读 | 30 分钟/本 | SCAN_NOTES.md |
| L2 通读 | 理解全书骨架和论证链 | 正常速度 | reading_notes.md (章节摘要) |
| L3 精读 | 提取方法论单元 | 慢速 + 标注 | 交给阶段 1 的 extractor |

**关键**: L2 通读产出的 `reading_notes.md` 是阶段 1 所有 extractor 的**共享上下文**, 避免 5 个 agent 各读一遍全书。

## Adler 四步 (前三步来自 Adler, 第四步是本 skill 新增)

### 步骤 1 — 结构 (Structural)

识别书的"骨架",回答:

- **这本书属于什么类型?** (方法论 / 传记 / 哲学 / 实操手册 / ...)
- **它的主旨用一句话说是什么?** — 必须真的能压缩到一句
- **它的主要部分如何组合成一个整体?** — 列出 3–7 个一级论点并标出它们的关系 (并列 / 递进 / 对比 / 反驳)
- **作者试图解决的核心问题是什么?**

### 步骤 2 — 解释 (Interpretive)

- **关键术语**: 列出作者反复使用、有特定含义的概念词,给每个词写一句"作者本人的用法"定义 (不是字典定义)
- **核心命题**: 用你自己的话重述作者的 5–15 个核心主张
- **论证链**: 这些主张之间是怎么推导的? 作者用什么证据支撑?

### 步骤 3 — 批判 (Critical) ★ 最容易被跳过也最重要

Adler 原话: "在没有找出论证中的错误之前,你不能不同意作者。" 反过来: **在找出作者的局限之前,你也不能完全同意。**

必须回答:
- **作者的时代局限**: 这本书写于什么时期? 哪些前提可能已经不成立?
- **作者的立场盲点**: 作者的身份 / 行业 / 文化背景让他忽略了什么?
- **未被证明的假设**: 作者把什么当成了不言自明但其实需要论证的东西?
- **反对意见**: 如果有人要反驳这本书,最强的论点会是什么?

这一步的产出会直接成为每个 skill 的 **Boundary (B)** 字段来源。

### 步骤 4 — 应用潜力 (Applicability, 本 skill 新增)

- **哪些内容可以 skill 化?** — 框架 / 清单 / 原则 / 决策程序
- **哪些内容不适合 skill 化?** — 纯史料 / 纯故事 / 纯情感 (但可作为其他 skill 的 example)
- **预估 skill 数量**: 给一个粗略区间 (不要硬凑)
- **预估优先级**: 从"最能赋能普通人"的角度给候选 skill 排序

## 质量门 (进入阶段 1 之前必须满足)

- [ ] 主旨能用一句话说清
- [ ] 骨架列出 3–7 个一级论点
- [ ] 关键术语词典有 ≥5 条
- [ ] 批判阶段至少列出 3 条作者局限 (这一步没做到位就不能继续)
- [ ] 已向用户展示 BOOK_OVERVIEW.md 并得到确认

## 常见失败模式

1. **跳过批判阶段** — 导致 skill 把作者的偏见当真理
2. **骨架不是"作者的骨架",而是自己的想法** — 注意你在写摘要还是在写读后感
3. **术语定义用字典 / 常识而非作者的特定用法** — 作者用"能力圈"和字典用"能力圈"不是一回事

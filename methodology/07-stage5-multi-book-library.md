# 阶段 5 — 多书协同层 (Library Layer)

## 目标

把多本独立蒸馏的 skill pack 聚合成一个**跨书的知识网络**,让不同书的方法论能在同一个决策场景下协同被调用,而不是各自孤岛。

这是 book2skill 多书模式的**核心差异化层**。没有这一层,10 本书只是 10 个独立的 skill 仓库,用户面对决策时还是不知道该调哪个。

## 何时进入此阶段

- 用户明确说"多本书" / "批量" / "融合打通" / "形成一个知识体系"
- 已完成至少 2 本书的单本流水线 (阶段 0–4)
- 用户确认要建立跨书知识网络

## 产出结构

```
library/
├── LIBRARY_OVERVIEW.md          # 知识库主旨、覆盖领域、书单、整体架构
├── THEME_INDEX.md               # 跨书主题聚合 (如"风险判断"主题下有哪些 skill)
├── DECISION_INDEX.md            # ★ 决策场景反向索引 (见阶段 6)
├── GLOSSARY_UNIFIED.md          # 跨书统一术语词典 (解决术语冲突)
├── CONFLICTS.md                 # 跨书冲突记录 (同一主题不同作者的不同观点)
├── COMPARISON_MATRIX.md         # 同主题方法论对比矩阵 (见阶段 6)
├── DECISION_CARDS/              # 决策卡片 (见阶段 6)
│   ├── pricing-decision.md
│   ├── market-entry.md
│   └── ...
└── books/
    ├── <book-1-slug>/           # 原有的单本结构
    ├── <book-2-slug>/
    └── ...
```

## 执行步骤

### 步骤 1 — 建立知识库骨架 (LIBRARY_OVERVIEW.md)

读取所有已完成单本的 `BOOK_OVERVIEW.md`,聚合产出:

- **知识库主旨**: 这批书合在一起,覆盖了什么领域? 能回答什么类型的决策问题?
- **书单与定位**: 每本书在知识库中的角色 (基础理论 / 实操方法 / 哲学视角 / 政策背景 / ...)
- **覆盖矩阵**: 哪些决策领域覆盖充分? 哪些有缺口?
- **推荐使用顺序**: 先读哪本建立基础, 再读哪本补充视角

模板: `templates/LIBRARY_OVERVIEW.md.template`

### 步骤 2 — 跨书主题聚合 (THEME_INDEX.md)

把所有书的所有 skill 按主题重新组织,不按书分组。

主题维度示例 (商业决策导向):
- 风险判断与不确定性
- 价值评估与定价
- 市场进入与竞争策略
- 资源配置与优先级
- 时机判断与周期
- 人性与行为偏误
- 系统思维与因果链
- 长期主义 vs 短期取舍

每个主题下列出:
- 来自哪些书的哪些 skill
- 这些 skill 之间的关系 (互补 / 对比 / 递进 / 冲突)
- 推荐组合调用顺序

### 步骤 3 — 统一术语词典 (GLOSSARY_UNIFIED.md)

合并所有单本的 `glossary.md`,解决跨书术语冲突:

- **同一术语不同定义**: 如"价值"在曼昆经济学、炒股的智慧、易经中含义不同 → 显式记录差异,不强行统一
- **不同术语同一概念**: 如曼昆的"机会成本"和段永平的"不做清单"底层相关 → 建立映射
- **新增跨书概念**: 蒸馏过程中涌现的、任何单本都没有但组合后才出现的新概念

输出格式:
```yaml
- term: 价值
  unified_definition: |
    (跨书综合定义,说明在不同语境下的不同含义)
  per_book_usage:
    - book: 经济学原理(曼昆)
      usage: "主观效用 + 客观交换价值的混合"
    - book: 炒股的智慧
      usage: "企业未来现金流的折现"
    - book: 易经的智慧
      usage: "事物的本质属性与时位关系"
  key_insight: |
    三者看似不同,但都在回答"这个东西值什么"。
    曼昆回答"市场认为它值多少",炒股回答"未来它值多少",
    易经回答"它本质上处于什么状态"。
    做商业决策时需要同时考虑三个维度。
```

### 步骤 4 — 冲突记录 (CONFLICTS.md) ★

**这是多书融合最有价值的产出。** 不同作者对同一问题有不同观点,不强行统一,而是显式记录冲突和适用条件。

冲突类型:
1. **方法论冲突**: 曼昆的"理性人假设" vs 易经的"变易无常" → 适用条件不同
2. **价值取向冲突**: 共同富裕的"公平优先" vs 炒股的智慧的"效率优先" → 决策情境不同
3. **时间尺度冲突**: 短期套利 vs 长期价值 → 投资期限不同

每条冲突记录:
```yaml
- conflict_id: C01
  topic: 对"风险"的理解
  positions:
    - book: 经济学原理(曼昆)
      position: "风险 = 不确定性,可量化,用概率分布描述"
    - book: 炒股的智慧
      position: "风险 = 亏损可能性,核心是本金安全"
    - book: 易经的智慧
      position: "风险 = 时位不当,吉凶悔吝因时势而变"
  resolution: |
    不统一。三者适用于不同决策层级:
    - 曼昆: 适用于可量化的金融决策
    - 炒股: 适用于个人投资的本金保护
    - 易经: 适用于战略级时机判断
  decision_guidance: |
    做商业决策时,先用曼昆框架量化风险,
    再用炒股框架检查下行保护,
    最后用易经框架判断时机是否得当。
```

### 步骤 5 — 跨书关系集中维护 (不回填单本) ★

**设计决策**: 跨书 related_skills **不回填到单本 SKILL.md**,而是集中在 library/ 层维护。

**原因**:
- 单本 SKILL.md 保持纯净,代表"这本书本身的产出"
- 跨书关系是多书协同层的产物,属于 library/ 的职责
- 避免单本 SKILL.md 被反复修改,失去版本稳定性
- 跨书关系在 THEME_INDEX.md 和 DECISION_INDEX.md 中自然体现

**维护方式**:
跨书关系在 `THEME_INDEX.md` 的每个主题条目中记录:

```yaml
- theme: 逆向思维与反转判断
  skills:
    - book: 穷查理宝典
      skill: inversion-thinking
      role: 核心方法
    - book: 毛选
      skill: maodun-fenxi
      role: 补充视角 (找主要矛盾)
    - book: 易经的智慧
      skill: jiu-yao-bianzheng
      role: 哲学根基 (阴阳转化)
  cross_book_relations:
    - from: inversion-thinking
      to: maodun-fenxi
      relation: composes-with
      note: "逆向思维找失败因素 + 矛盾分析找主要矛盾 = 决策双检"
    - from: inversion-thinking
      to: jiu-yao-bianzheng
      relation: depends-on
      note: "逆向思维是阴阳转化的具体应用"
```

单本 SKILL.md 的 `related_skills` 字段**只填单本内的链接**,跨书链接在 library/ 层查。

## 不变量

1. **不强行统一冲突**: 不同作者的观点差异是知识,不是错误。记录 + 给适用条件,不选边站。
2. **跨书链接必须有 note**: 说明为什么这两个 skill 要跨书关联,关联的价值是什么。
3. **术语词典保留 per_book_usage**: 不抹平差异,差异本身就是决策依据。
4. **THEME_INDEX 按决策主题组织,不按书组织**: 用户面对的是决策,不是书。

## 常见失败模式

1. **把多书协同做成"书单汇总"** — LIBRARY_OVERVIEW 写成了书评集,没有跨书整合。要求: 必须有覆盖矩阵和主题聚合。
2. **冲突记录选边站** — 试图证明某个作者"更对"。要求: 记录差异 + 给适用条件,不评判。
3. **跨书链接硬凑** — 为了凑数给不相关的 skill 建立链接。要求: 每个跨书链接必须有 note 说明价值。
4. **术语统一抹平差异** — 把不同作者的概念强行合并成一个定义。要求: 保留 per_book_usage,差异是知识。

# 阶段 6 — 决策导向产出 (Decision-Oriented Output)

## 目标

把多本书的 skill 网络转成**可以直接用于商业决策的工具集**。不是"这里有哪些 skill"的正向索引,而是"我遇到决策 X,该调用哪些 skill 组合"的反向索引。

这是 book2skill 多书模式的**最终交付层**。前面的所有阶段都在"造零件",这一层把零件组装成"决策工具箱"。

## 何时进入此阶段

- 已完成阶段 5 (多书协同层)
- 用户明确了决策场景类型 (本 skill 默认商业决策,可配置)
- 用户想要"指导决策"而不仅是"知识库"

## 三类产出

### 产出 1 — DECISION_INDEX.md (决策场景反向索引) ★

**核心文件。** 按决策场景组织,不按书、不按主题、不按 skill 组织。

用户面对的是决策,不是书。所以索引的第一维度必须是"决策场景"。

商业决策场景分类 (可根据用户实际需求调整):

| 一级分类 | 二级场景示例 |
|---|---|
| 投资决策 — 价值判断 | 估值/内在价值/安全边际/泡沫识别 |
| 投资决策 — 时机判断 | 择时买入/择时卖出/拐点识别 |
| 投资决策 — 仓位管理 | 仓位大小/加仓减仓/集中分散 |
| 投资决策 — 风险控制 | 止损/对冲/下行保护/黑天鹅预案 |
| 战略决策 | 进入/退出/转型/多元化/聚焦 |
| 定价决策 | 新品定价/调价/折扣/溢价 |
| 竞争决策 | 跟随/对抗/差异化/低成本 |
| 资源配置 | 优先级/预算分配/人力投入/机会成本 |
| 周期判断 ★ | 周期定位/趋势识别/反转信号/跨周期配置 |
| 不确定性决策 ★ | 模糊情境/信息不全/多变量耦合/范式转换 |
| 风险决策 | 对冲/承担/转移/规避/尾部风险 |
| 组织决策 | 招聘/解聘/分权/激励/文化 |
| 分配决策 ★ | 利益分配/公平效率权衡/短期长期平衡 (政策类书特有) |

每个决策场景的索引条目:
```yaml
- decision: 定价新产品
  scenario: "公司要给一款新产品定价,不确定该走高端还是性价比路线"
  skill_combination:
    - skill: value-assessment (穷查理宝典)
      role: 评估产品对客户的真实价值
    - skill: opportunity-cost (段永平)
      role: 考虑不选另一路线的代价
    - skill: supply-demand (曼昆经济学)
      role: 判断市场均衡价格区间
    - skill: shi-positioning (易经的智慧)
      role: 判断产品在市场中的"时位"
  decision_flow: |
    1. 先用 value-assessment 量化产品对目标客户的价值
    2. 用 supply-demand 找到市场可接受的价格区间
    3. 用 opportunity-cost 检查不同定价路线的取舍
    4. 用 shi-positioning 判断当前时机适合高端还是性价比
  anti_patterns:
    - "只看成本加成定价 (忽略价值)"
    - "只看竞品定价 (忽略差异化)"
    - "只看当前需求 (忽略周期)"
  source_conflicts:
    - conflict: C01 (曼昆 vs 易经对"风险"的理解)
      relevance: "定价新产品本质是在不确定性下决策,需同时考虑量化风险和时位"
```

### 产出 2 — DECISION_CARDS/ (决策卡片)

一页纸压缩版,用于决策时快速查阅。不是完整 SKILL.md,而是**决策场景维度的压缩视图**。

每张卡片对应一个决策场景,不是对应一个 skill。一张卡片可能组合多个 skill 的精华。

模板: `templates/DECISION_CARD.md.template`

卡片内容 (极简):
- 决策场景一句话描述
- 核心问题 (这个决策真正在问什么)
- 3-5 个关键步骤 (来自组合 skill 的 E 段精华)
- 3-5 个反模式 (来自组合 skill 的 B 段精华)
- 关键术语 (来自 GLOSSARY_UNIFIED 的相关条目)
- 深入链接 (指向相关 SKILL.md)

### 产出 3 — COMPARISON_MATRIX.md (对比矩阵)

同一决策主题下,不同作者/书的方法论对比。

帮助用户理解"什么时候用哪个版本",而不是"哪个对哪个错"。

矩阵格式:
```markdown
| 决策维度 | 曼昆经济学 | 炒股的智慧 | 易经的智慧 | 共同富裕 |
|---|---|---|---|---|
| 核心问题 | 资源如何最优配置 | 本金如何安全增值 | 时位如何判断 | 利益如何分配 |
| 决策单元 | 个人/企业 | 个人投资者 | 事物整体 | 社会整体 |
| 时间尺度 | 中短期 | 中长期 | 长周期 | 跨代际 |
| 风险观 | 可量化不确定性 | 亏损可能性 | 时位不当 | 系统性失衡 |
| 关键变量 | 价格/数量 | 价值/价格 | 阴阳/时位 | 效率/公平 |
| 适用场景 | 市场交易决策 | 个人投资决策 | 战略级时机判断 | 政策合规判断 |
| 局限 | 假设理性人 | 个人视角 | 难以量化 | 价值取向特定 |
```

## 执行步骤

1. **收集决策场景**: 从用户实际需求 + 阶段 5 的 THEME_INDEX 中提取
2. **场景到 skill 映射**: 对每个决策场景,找出哪些 skill 能提供方法支持
3. **设计组合调用流程**: 不是简单罗列 skill,而是给出调用顺序和每步作用
4. **写决策卡片**: 把组合流程压缩成一页纸
5. **建对比矩阵**: 对关键决策维度,列出不同书的方法论差异
6. **用户验证**: 拿 2-3 个真实决策场景让用户走一遍,看流程是否跑得通

## 质量门

- [ ] DECISION_INDEX 覆盖用户明确的决策场景类型 (商业决策)
- [ ] 每个决策场景至少组合 2 个 skill (单 skill 解决的决策不需要卡片)
- [ ] 每张决策卡片有至少 3 个反模式 (来自 B 段)
- [ ] COMPARISON_MATRIX 至少覆盖 5 个核心决策维度
- [ ] 已用真实场景验证 (用户走完 2-3 个决策流程)

## 常见失败模式

1. **DECISION_INDEX 退化成 THEME_INDEX** — 按主题分组而不是按决策场景分组。要求: 第一维度必须是"用户面对的决策"。
2. **决策卡片太长** — 超过一页纸。要求: 卡片是压缩视图,细节去 SKILL.md 看。
3. **组合调用没有顺序** — 只列 skill 列表不给流程。要求: 必须有 decision_flow,说明调用顺序和每步作用。
4. **对比矩阵选边站** — 试图证明某个作者的方法更优。要求: 记录差异 + 给适用条件。
5. **决策场景太抽象** — "如何做投资决策" 不是场景,"是否应该在当前估值买入某股票" 才是。要求: 场景必须具体到可识别。
